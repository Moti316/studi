## סכמת שורת-מטא מורחבת (ממשל-v2 · תוצר-3)

כל תת-משימה **פתוחה** נושאת **שורת-מטא** מתחתיה — אומדן-מאמץ + סיכון + בעלות + שער-אימות. כך
ענף-הבקרה (`oversight` — בהקמה, סשן-B) יכול לבדוק את ההתקדמות מול **checklist אובייקטיבי** ולא מול תחושה.

```
- [ ] **<ID> · <כותרת>** — <תיאור> · קריטריון-קבלה: … · ref: …
  - 📊 **מטא:** ⏱<h>h · 🤖<n>(<slugs>) · 💲~$<range> · 🟢|🟡|🔴 · ראש-צוות:<lead> · <🚩דורש-מוטי|—> · אימות:<Workflow|solo>
```

**מקרא השדות:**

| שדה              | משמעות                                                                                                    |
| ---------------- | --------------------------------------------------------------------------------------------------------- |
| ⏱`<h>h`          | אומדן-מאמץ בשעות (גס; לכיול בעת-הביצוע)                                                                   |
| 🤖`<n>(<slugs>)` | מספר-הסוכנים שצפויים לגעת + ה-slugs שלהם (מתוך הרוסטר — [`../../teams/README.md`](../../teams/README.md)) |
| 💲`~$<range>`    | טווח-עלות-AI צפוי (`$0` = קוד/תיעוד; חריגה מ-hard-cap = דגל-בקרה)                                         |
| 🟢/🟡/🔴         | סיכון: 🟢 שגרתי · 🟡 עלות/אינטגרציה/ציות · 🔴 כספי/אבטחה/רגולציה                                          |
| ראש-צוות         | ה-lead האחראי: `strategic-lead` · `builder-lead` · `quality-lead` · `coordinator-lead`                    |
| 🚩דורש-מוטי      | חוסם על אישור-מוטי מפורש (`—` = לא-חוסם)                                                                  |
| אימות            | `Workflow` (code-review/security-review) למשימות-קוד מהותיות · `solo` לתיעוד/החלטה                        |

> **שימוש-הבקרה (סשן-B):** השדות הם ה-checklist האובייקטיבי של ענף-הבקרה:
> ✅-ללא-`activity-log` → דגל · 🚩-ללא-אישור-מוטי → צו-עצירה · חריגת-💲/🔴-ללא-`Workflow` → דגל.
> תתי-משימות **שהושלמו** (✅) נושאות מטא-מינימלי (✅ + ראש-צוות) — לא רטרואקטיבי-מלא.
