> **שימוש**: כל chunk/scenario שמיובא ל-StudiBuilder מסומן `in_scope: true|false` ע"פ הרשימה כאן.

## הערת-וועדה (קריטית)

> "בוועדת ההסמכה עליך לדעת **להפנות** לתקנה הרלוונטית, **להסביר** את הרציונל שמאחוריה, ו**ליישם** אותה באופן מעשי על תרחישים שונים."

3 פעולות-מבחן שצריך לאמן:

| פעולה                                  | Quiz type מתאים            |
| -------------------------------------- | -------------------------- |
| Identification (איזה חוק/תקנה רלוונטי) | MCQ-long, MatchingPairs    |
| Explanation (rationale מאחורי התקנה)   | ExplanationCard, MCQ-short |
| Application (החלה על תרחיש)            | ScenarioWalkthrough        |

## Implementation Notes

### Scope-filter ב-import pipeline

כל chunk שמיובא מ-Drive/megen חייב לכלול metadata:

```yaml
in_scope: true | false
scope_refs:
  - id: '1.1' # תקנות ארגון הפיקוח (ממונים)
    section: 'סעיף 3' # אופציונלי
  - id: '5.3' # ת"י 31010
status: '[מאומת]' # | '[מוסקנא]' | '[לא ידוע]'
```

**כלל-מתחייב**:

- `in_scope=true` → eligible לשאלות-וועדה
- `in_scope=false` → נשמר כ-reference במנגנון אך לא עולה ל-quiz בפועל
- `[לא ידוע]` חסום בכל מקרה (כלל מ-ADR-005)

### Coverage tracker

מסך admin: `/admin/coverage` — מציג לכל אחד מ-57 הפריטים:

- מספר questions שיש (יעד: ≥ 5 per ID)
- מספר scenarios (יעד: ≥ 2 per ID)
- אחוז recall של מוטי

מבטיח שאף ID לא נופל בין הכיסאות לפני הוועדה.

### Out-of-Scope Content — תוכן שמסומן in_scope=false

תוכן שזוהה ב-Drive אבל לא ב-57:

- **חוזק חומרים + תהליכי-ייצור** (טל גרינהויז) — חלק מ-curriculum (פרק ד.36) אבל לא נושא-וועדה ישיר → in_scope=mixed (נשמר ל-deep-explanation בלבד)
- **מערכות אוטומטיות + רובוטים** — חלק מ-curriculum (פרק ד.38) → in_scope=mixed
- **ריתוך, עיבוד שבבי** (מיכאל פייזל) — חלק מ-מיגון-מכונות → in_scope=true תחת 2.0 (פקודה פרק ג)
