## 5.1 — ת"י 45001 (ISO 45001) — מערכת ניהול בטיחות ובריאות תעסוקתית

> **מקור**: PDF "תקן ניהול בטיחות ובריאות 45001 - סיכום שיעור" (אבי גריפל, Drive 2026-05-20)
> **File ID**: `1u8bJMpJ1L5UZOksT2P9eFU7S9xg87Rjf`

### מהות

ת"י 45001 הוא **תקן בינלאומי** למערכת-ניהול בטיחות ובריאות תעסוקתית. **לא חובה** משפטית, אבל **אם המפעל פועל לפיו** — זה משדרג את הבטיחות, מראה רצינות, ומגן משפטית במקרה של תאונה. גם יתרון-שיווקי: במכרזים יש מקומות שלא יקבלו הזמנה ממפעל בלי תקן.

### היסטוריה ויחס לתקנות-ישראליות

- התקן עצמו מ-שנות ה-90 (כ-OHSAS 18001, ואחר-כך 45001 ב-2018)
- **תקנות-ישראליות לתכנית-לניהול-בטיחות 2013** (1.2 ב-content-scope) מבוססות על התקן
- המודל: **מדיניות → תכנון → ביצוע → מבדק → שיפור** (PDCA)

### דרישות-מפתח

1. **סקר-הנהלה** (Management Review) — שנתי/חצי-שנתי/רבעוני. ההנהלה סוקרת:
   - האם עמדנו בתכנית-הבטיחות?
   - כיוונים להמשך
   - התקן **מחייב** סקר-הנהלה (חשוב — מבטיח שהנושא יגיע ל-top management)
2. **תיעוד-אחריות** — האחריות על קיום-התקנות חלה על **תופש-המקום** (בעל-המפעל, מבצע-בנייה, וכו')
3. **4 נהלי-חובה** (לפי התקנות-2013):
   - נוהל הדרכה
   - נוהל תגובה בחירום
   - נוהל הכנסת גורם-סיכון חדש
   - נוהל זיהוי, איתור וסילוק מפגעים

### תהליך-קבלת-תעדה (Certification)

1. **כתיבת התקן** עצמו: ע"י מכון-תקנים בינלאומי (לדוגמה NEN בהולנד), עם נציגי תעשיות + מכוני-תקינה. תהליך 3-4 שנים.
2. **חיבור-עצמי** ע"י המפעל: ~₪250-500K (תשלום למומחים). או קניית תקן-בריטי ב-~₪800.
3. **התעדה (Certification)**: מכון-התקנים בא כל-שנה לחדש את התעודה.

## 5.6 — OSHA Hierarchy of Controls (מדרג-בקרות)

> **מקור**: אותו PDF (תקן 45001 — אבי גריפל). בנוסף — מוזכר ב"חזרה להסמכה 2" עמ' 22-23.

### מהות

**היררכיית-הבקרות** היא סדר-העדיפות לבחירת-בקרות להפחתת-סיכונים. הסדר משקף **יעילות יורדת**: בקרות-עליונות הכי-יעילות, בקרות-תחתונות הכי-חלשות.

### 6 שכבות (מלמעלה למטה — מהיעיל-ביותר למיותר-ביותר)

1. **סילוק (Elimination)** — סילוק-מוחלט של גורם-הסיכון
   - דוגמה: החלפת חומר-מסוכן בחומר לא-מסוכן
2. **הפחתה (Substitution / Reduction)** — הפחתת-עוצמה של גורם-הסיכון
   - דוגמה: הורדת-טמפרטורה בתהליך-עבודה כדי שלא יגרום-כווייה
3. **מיגון הנדסי (Engineering Controls)** — חציצה-פיזית בין-העובד לסיכון
   - דוגמה: גידור-מכונה, מיגון-אוטומטי, אבטחת-הרמה
4. **בקרות מנהלתיות (Administrative Controls)** — שליטה-על-התנהגות
   - דוגמה: הדרכה, נהלים, פיקוח, שילוט, חוזה-עבודה
5. **ציוד מגן אישי (PPE)** — מיגון על-גוף-העובד עצמו
   - דוגמה: קסדה, רתמה, מסכת-נשימה
6. **עזרה ראשונה + טיפול רפואי** — מתי שכבר קרה נזק
   - דוגמה: מילוי-קופסת עזרה-ראשונה, אחראי-עזרה-ראשונה

### תובנה-חשובה לוועדה

- "**הכי פחות עדיף להגיע למצב שצריך להשתמש בעזרה ראשונה**" — תמיד נעדיף קודם את ה-מה-שראשון בהיררכיה.
- אם סילקנו את המפגע — נשמח שהציוד-עזרה-ראשונה יעלה אבק (כי לא צריכים אותו)
- ההיכרכה היא **עולמית** (OSHA). בישראל יש מקבילה: **נ"קי** = ניתוח-קבלה-יישום של הבקרות (3 שלבים)

### הקשר נ"קי (ישראלי) ↔ ה-Hierarchy (עולמי)

נ"קי = **ניתוח שיטות → קבלת החלטה → יישום הבקרה**.

הוחלף ע"י המחוקק כי הבקרות-המנהלתיות (שילוט, הדרכה) לא היו יעילות-מספיק. מנהל-הבטיחות רצה לדחוף את הממונה לבחור בקרות-טובות-יותר — מעבר ל-בקרות-זולות-חלשות. השיטה מעודדת לבדוק את **כל האפשרויות** לפני בחירת-האופציה-הזולה-ביותר.

### עיקרון-נלווה: ALARP

**As Low As Reasonably Practicable** — שאיפה להוריד את רמת-הסיכון **כמה-שניתן באופן-סביר** (לא תמיד עד-אפס, אלא עד-המעשי-כלכלית).

## 6.3 — HAZOP (Hazard and Operability Studies)

> **מקור**: אותו PDF (תקן 45001 — אבי גריפל)

### מהות

**HAZOP (הזופ)** = שיטת ניהול-סיכונים למתקנים-תהליכיים (process plants) — לדוגמה בתי-זיקוק, מאפיות, מעבדות-כימיות. בודקת **סטיות מתפקוד-תקין** במתקן.

### מתי משתמשים

**בטיחות תהליכית (Process Safety)** — שונה מבטיחות-בעבודה-רגילה:

|             | בטיחות-עבודה-רגילה | בטיחות תהליכית                             |
| ----------- | ------------------ | ------------------------------------------ |
| **שכיחות**  | גבוהה              | נדירה                                      |
| **חומרה**   | נמוכה-בינונית      | **קטסטרופלית** כשקורה (פצצה, רעלים, רעידה) |
| **דוגמאות** | נפילה, חיתוך-יד    | פיצוץ-כלי-לחץ, דליפת-גז                    |

### כלי-עזר: P&ID

**P&ID** (Piping and Instrumentation Diagram) = תרשים-זרימה של המתקן.

- **P** = pipes (צנרת)
- **I** = instruments (אביזרים)
- **D** = diagram

ה-HAZOP מבוסס על קריאת P&ID + שאילת-שאלות **"מה אם..."** על כל-נקודה.

### תהליך-העבודה — Guide Words

צוות-HAZOP עובר על כל-נקודה במתקן ומחיל **מילות-מנחה** (guide words):

- **No** / **Not** — מה אם אין-זרימה?
- **More** — מה אם יש יותר-מ-25-מעלות?
- **Less** — מה אם פחות-לחץ-מהרגיל?
- **Reverse** — מה אם זרימה-אחורנית?
- **Other than** — מה אם חומר-שגוי במערכת?

לכל סטייה → ניתוח: **סיבות אפשריות + השלכות + בקרות-קיימות + בקרות-חסרות**.

### תקן-מקור

- **IEC 61882** — Hazard and Operability Studies (HAZOP) — תקן בינלאומי
- ב-content-scope.md זה **5.5**

### מי מבצע HAZOP?

לפי PDF: לא הממונה-בטיחות בעצמו! צוות-HAZOP = **מהנדסים + מומחים-טכניים** שמבינים את התהליך-עומק. הממונה-בטיחות **מגיע ואומר שצריך לבצע HAZOP** — אבל לא הוא שמוביל את הניתוח-עצמו.

### דוגמת-שאלה אפשרית

- "מה ההבדל בין בטיחות תהליכית לבטיחות עבודה רגילה? תן דוגמה."
- "מי אחראי לבצע HAZOP במפעל?"

## 6.1 — JSA (Job Safety Analysis) — ניתוח-בטיחות-משימתי

> **מקור**: עבודת-גמר "ניהול הסיכונים באתר בנייה - הערכת סיכונים בשיטת JSA" (לוי יצחק - דנה, 2025-12-01, מנחה: בני סלוטקי)
> **קובץ**: עבודת-גמר 28 עמודים — נשלח ע"י motilev8 כדוגמת-מעשית

### מהות

**JSA (Job Safety Analysis)** = שיטה ל-**ניתוח-בטיחות-משימתי** — מפרקים מטלה ספציפית לשלבי-ביצוע, ולכל-שלב: גורם-סיכון → הערכה → בקרה → סיכון-שיורי.

### מתי משתמשים

- לעבודות **בסיכון-גבוה** (חיפוי-אבן בקירות-פיתוח, פיגומים, חפירות)
- כאשר תפעולית-יומיומית — לא חד-פעמית
- לפני **תחילת-עבודה חדשה**

### מבנה JSA — 11 עמודות (סטנדרט-ישראלי)

| #   | עמודה                             | הסבר                   |
| --- | --------------------------------- | ---------------------- |
| 1   | אזור                              | מיקום פיזי באתר        |
| 2   | עמדת-העבודה / שלב-בהליך           | פירוט-מטלה             |
| 3   | ציוד                              | מה משתמשים בו          |
| 4   | גורמי-סיכון                       | מה עלול לגרום נזק      |
| 5   | תרחיש-התממשות                     | "אם X אז Y"            |
| 6   | הערכת-סיכון (לפני) — סבירות (1-4) | מטריצת-החלטה           |
| 7   | הערכת-סיכון (לפני) — חומרה (1-4)  | מטריצת-החלטה           |
| 8   | רמת-סיכון = ס × ח                 | ירוק/צהוב/אדום         |
| 9   | בקרות נדרשות — הנדסיות            | מיגון פיזי             |
| 10  | בקרות נדרשות — מנהלתיות           | נהלים, הדרכה, פיקוח    |
| 11  | בקרות נדרשות — צמ"א (PPE)         | קסדה, נעלי-בטיחות וכו' |
| 12  | הערכת-סיכון (אחרי בקרות)          | סבירות + חומרה חדשים   |
| 13  | אחראי לביצוע                      | מנהל-עבודה / ראש-צוות  |

### דוגמה — חיפוי-אבן בקירות-פיתוח

- **גורם-סיכון**: דריסה / התחשמלות / נפילת-מנוף
- **לפני בקרות**: ס=3, ח=4 → **12 (אדום, לא קביל)**
- **בקרות**:
  - **הנדסיות**: רישיון-הנדסי, התקנת-תאורה, מערכת-יציב + בקרת-קלה
  - **מנהליות**: הדרכה לפי שיטת **"נעילה ותיוג"** (LOTO), הדרכת-עובדים
  - **צמ"א**: קסדה + כפפות + ווסט-זוהר + **נעלי-בטיחות S3**
- **אחרי בקרות**: ס=1, ח=4 → **4 (צהוב, ניתן להמשיך)**

### תהליך JSA — 4 שלבים

## 6.2 — FMEA / FMECA (Failure Modes & Effects [+Criticality] Analysis)

> **מקור**: מצגת "תקנות תכנית-לניהול-בטיחות 2013 — יום 3" (אבי גריפל, נובמבר 2013)
> **File ID**: `19YqDBWOuia0MqH-WHUNqdRZ3JFdESdQp`

### מהות

**FMEA** = ניתוח אופני-כשל ותוצאות. **FMECA** = + הוספת **קריטיקליות** (חומרת-הכשל).

טכניקה לזיהוי-שיטתי של:

1. כל **אופני-הכשל** של מערכת (failure modes)
2. **ההשפעה** של הכשלים (effects) על המערכת וסביבתה — עם סיווג הסתברות + חומרה
3. **פעולות-מניעה** ו/או צמצום-נזק

### איפה משתמשים

|     | רכיב                                |
| --- | ----------------------------------- |
| 1   | **חומרה** (hardware) — מרכיבי-מערכת |
| 2   | **תפקודי-מערכת**                    |
| 3   | **התנהגות-מפעילים**                 |
| 4   | **פקודות תוכנת-מחשב**               |
| 5   | **אחזקה**                           |
| 6   | **גריטה / פירוק**                   |

### 8 שלבי-ביצוע

1. סקירה כללית של מרכיבי-המערכת ותפקודיהם
2. סקירת כל המשימות שהמערכת מבצעת
3. **זיהוי כל אופני-הכשל** לכל מרכיב
4. קביעת **סיבות לכשל** + תרחישי-כשל
5. קביעת **אוכלוסיות-יעד** שעלולות להיפגע (מערכת/אנשים/ציוד/סביבה)
6. קביעת **אפקט הכשל** על מרכיבים-אחרים + הסתברות + חומרה
7. קביעת **רמת-קריטיקליות** של כשל
8. **המלצה על שינויים** לצמצום הסתברות-לכשל ונזקיו

### גליון-עבודה (לפי NASA)

| תיאור-מרכיב | תפקודו | אופן-כשל | סיבות-לכשל | השפעת-כשל | פעולה מתקנת | דרגת קריטיות |
| ----------- | ------ | -------- | ---------- | --------- | ----------- | ------------ |

### מנגנוני-כשל אופייניים (לזכור)

|     | מנגנון                    |
| --- | ------------------------- |
| 1   | שבר / סדק / עייפות        |
| 2   | עומס-יתר אלקטרוני / בלאי  |
| 3   | זיהום / סגירה לא-הרמטית   |
| 4   | תגובה לחומר-כימי / חימצון |
| 5   | קרינה / שינויים-במבנה     |

### יתרונות

- **שיטתי + מקיף + יסודי** — כל הכשלים, כולל Single-Point-Failure
- מסייע לשיפור-עיצוב, מהימנות, תפעול
- מאתר **יחסי-גומלין** בין-כשלים

### חסרונות

- **יקר ודורש כוח-אדם** למערכות-מורכבות
- בלתי-אפשרי להעריך הסתברויות לכשלים-בו-זמניים

- בדרך-כלל לא-מתחשבים בטעויות-אנוש או תנאי-סביבה-קשים

## 6.3 הרחבה — HAZOP — 7 מילות-מנחה (Guide Words) + אסון בהופאל

> **מקור**: אותה מצגת (אבי גריפל יום 3, 2013) — משלים את 6.3 שכבר תועד

### 7 מילות-מנחה (Guide Words) — לזכור!

על כל **מקטע-פיזי** (לדוגמה: צינור-הזנה למיכל), על כל **פרמטר** (זרימה/לחץ/טמפרטורה), מפעילים את ה-guide-words:

| מילה                       | משמעות-מעשית                  | דוגמה                                           |
| -------------------------- | ----------------------------- | ----------------------------------------------- |
| **אין** (No)               | אין-זרימה / אין-לחץ           | אין זרימה קדימה כשצריכה להיות                   |
| **יותר מ-** (More)         | יותר מ-תכונה-פיזית            | יותר זרימה (כמות/קצב), טמפרטורה, לחץ            |
| **פחות מ-** (Less)         | פחות מ-תכונה-פיזית            | פחות-לחץ, פחות-טמפרטורה                         |
| **חלק מ-** (Part of)       | מרכיב-חסר / שינוי-יחסי-חומרים | תערובת שונה ממה-שצריך                           |
| **יותר מאשר** (As well as) | נוכחות גורמים-נוספים          | אדים, מוצקים, זיהומים (אוויר, מים, חומצות)      |
| **אחר מ-** (Other than)    | מה-עוד יכול-לקרות?            | התנעה, השבתה, שדרוג, אי-מתן-שירות-במועד, תחזוקה |
| **אחורנית** (Reverse)      | זרימה-אחורנית                 | אם חומר יכול לזרום-מהמחולל-למיכל                |

### תהליך-שאלות לכל סטייה

1. **האם יתכן** שהסטייה תקרה?
2. אם כן — **מה יכול לגרום** לה?
3. **האם המפעיל ידע** שיש סטייה?
4. **מה התוצאות** של הסטייה?
5. האם הן **מסוכנות** או **מפריעות** לתפעול?
6. אם כן — **האם ניתן למנוע** (שינוי-עיצוב / שיטת-עבודה)?
7. האם **רמת-הסיכון** (הסתברות × תוצאות) מצדיקה השקעה?
8. אם כן — קבע: **שינוי + אחראי-ביצוע + לו"ז + מעקב-ביצוע**

### דוגמה היסטורית: אסון **BHOPAL** (3.12.1984)

- דליפת **מתיל-איזוציאנט** ממיכל-אחסון בהודו
- **2,000 הרוגים**, אלפי נפגעים-לחיים
- תקלה: זיהום-המיכל במים + כלורופורם
- **לא בוצע HAZOP** — והבעיה החריפה כי 3 מערכות-הגנה כשלו במקביל:
  1. ספיחת-אדים — לא היה תקין
  2. שריפת-גזים-עודפים — לא פעל
  3. מערכת-קירור-מיכל — לא פעלה

→ הוכחה היסטורית-קלאסית למה HAZOP **קריטי** ב-תעשייה-כימית.

### Tips לביצוע HAZOP

- אל-תיסחף — לא לבזבז זמן על אירועים-שוליים בהסתברות-אפסית
- לא רק אמצעים-הנדסיים — לחשוב גם על שינוי-שיטות-עבודה / ביקורות
- לבצע **גם בעת שינוי-במערכת** (לא רק בעת-תכנון-ראשון)
- לא לסמוך רק על אינטואיציה — היא לא תחליף ל-HAZOP

## 6.6 — FTA (Fault Tree Analysis) — ניתוח עצי-כשל ⭐ חדש ל-content-scope

> **מקור**: אותה מצגת — אבי גריפל יום 3 (2013)
> **הערה**: נושא זה לא היה ב-content-scope המקורי. ייתכן שצריך להוסיף כ-**6.6** או להתייחס לזה כתת-נושא של 6.2 (FMEA-משלים).

### מהות

**FTA** = הטכניקה ה-**פופולרית-ביותר** לניתוח-כשלים והערכת-הסתברות להתרחשותם. במיוחד ב-תעשייה-כימית, דלק, ותעבורה-חללית.

מבוסס על **מודל-לוגי גרפי** של גורמי-כשל, עם **שערים-לוגיים** (וְ / אוֹ / שערי-מאורעות).

### תהליך-הניתוח

מתחיל ב-**אירוע-ראשי** (Top Event) — הכשל הראשי שרוצים-לנתח. ממוקם בראש "העץ".

המטרה: לברר **איך** ו-**באילו אופנים** (תרחישים) האירוע-הראשי יכול להתרחש, ו-**מהן ההסתברויות**.

### 4 סימולים-חשובים

| סימול                                              | משמעות                                              |
| -------------------------------------------------- | --------------------------------------------------- |
| **אירוע ראשי / ביניים** (Top / Intermediate Event) | הכשל-העיקרי או גורם-תורם                            |
| **שער "וְ" (AND gate)**                            | הכשל-מעליו מתקיים **רק אם כל הכשלים מתחת** מתקיימים |
| **שער "אוֹ" (OR gate)**                            | מספיק כשל-אחד-מתחת — והכשל-מעליו מתקיים             |
| **מאורע בסיסי** (Basic Event / "Initiator")        | גורם-יסוד שלא מנסים לפרק אותו לתת-גורמים            |

### 4 צעדים-לביצוע

1. **יצירת עץ-הכשל** — הגדרת כשל-ראשי + כל המאורעות-העלולים-לגרום
2. **חישוב הסתברויות** — לכל מאורע בעץ
3. **זיהוי קבוצות-חיתוך** (Cut Sets) — קבוצות-מאורעות שאם-כולם-יקרו, הכשל-הראשי קורה
4. **זיהוי קבוצות-נתיב** (Path Sets) — קבוצות-מאורעות שאם-אף-אחד-מהם-לא-מתרחש, הכשל-הראשי **לא** יקרה

### דוגמה: לחץ-יתר-בדוד

```
         לחץ יתר בדוד (0.005 לשנה)
                  AND
       ┌──────────┴───────────┐
   לחץ עולה (1)          כשל שסתום שחרור (0.005)
       OR
  ┌────┴────┐
טעות מפעיל   כשל וסת-לחץ
  (0.2)        (0.8)
```

חישוב: **0.005 = 1.0 × 0.005** (כי "וְ" = מכפלה)

### מטרות-הניתוח

1. איתור **נתיבי-כשל** (שילוב-אירועים)
2. **חישוב-הסתברויות** להתרחשות-אירוע-ראשי
3. איתור **נתיב בעל הסתברות-הגבוהה-ביותר**
4. איתור נתיבים עם **מספר-מינימלי** של אירועים
5. קביעת **פגיעוּת** (vulnerability) של המערכת

### יתרונות + חסרונות

| יתרונות                                    | חסרונות                                   |
| ------------------------------------------ | ----------------------------------------- |
| הסתברויות כמותיות למערכות-מורכבות          | מתייחס רק לכשל-ראשי-אחד (צריך לצפות מראש) |
| איתור נקודות-פגיעות + Single-Point-Failure | מסובך ליצור עץ למערכות-מורכבות            |
| מסייע לעיצוב-מחדש                          | קשה לאמוד הסתברויות (במיוחד אם יש תלות)   |
|                                            | קשה לצפות את כל המאורעות-הבסיסיים         |

## מודלים-תיאורטיים נוספים (לזכור!) — שייכים לקטגוריה 5 או 6

### מודל 5M (לתאונות + להחלטות על בקרות)

| M              | משמעות       | דוגמאות-קונקרטיות                      |
| -------------- | ------------ | -------------------------------------- |
| **Man**        | הגורם האנושי | טעות, הפרת-נוהל, חוסר-הדרכה            |
| **Machine**    | גורם הנדסי   | ציוד, חומר, מתקן, מכונה                |
| **Media**      | סביבה        | סביבת-עבודה, מזג-אוויר, פסיכו-סוציאלית |
| **Mission**    | המטלה        | הוראות-עבודה, מורכבות-המשימה           |
| **Management** | ניהול        | תקציב, נהלים, ניהול-סיכונים            |

**שימוש כפול**:

1. **חקירת תאונה** — מה תרם כל מרכיב לתאונה?
2. **בחירת בקרות** — איך לטפל בכל מרכיב כדי למנוע הישנות ("תמונת-ראי")

### מודל הגבינה השוויצרית של ריזון (Reason's Swiss Cheese)

**רעיון**: בכל מערכת יש **שכבות-הגנה מרובות**. כל שכבה היא "פרוסת-גבינה עם חורים". אם **חורים בכל השכבות נפתחים יחד** — האנרגיה-המסוכנת עוברת ופוגעת.

### 5 שכבות-בקרה (לפי ריזון)

| שכבה                    | תפקיד           | דוגמאות                      |
| ----------------------- | --------------- | ---------------------------- |
| **מנחות** (Guiding)     | מכוונות-התנהגות | נהלים, הוראות-עבודה          |
| **מונעות** (Preventive) | חוסמות פיזית    | גידור-לבטח, בידוד, כבל-אבטחה |
| **מגבילות** (Limiting)  | מתריעות         | התראות, אזעקה                |
| **מגיבות** (Reactive)   | מצמצמות-נזק     | כיבוי-אוטומטי, שסתום-פריצה   |
| **משפרות** (Improving)  | מונעות-הישנות   | ריענון-הדרכה, מבדקים         |

### היררכיית-שיטות (לסיכום!)

**שיטות-בסיסיות** (נלמדו ביום-שני):

- **PHA** (Preliminary Hazard Analysis) — ניתוח-סיכונים ראשוני

III. המלצות         II. זיהוי         I. שחזור

| **מיידיים** | הגורמים-הישירים שגרמו לתאונה                                                        | סדק בצינור                      |

## 8.1 — שיטת 5Y (5 Why's) — חמישה צעדים לאחור

### מהות

**שיטה אינדוקטיבית** לאיתור גורמים-שורשיים. שואלים "**למה?**" חזרה-חזרה (לפחות 3 פעמים) עד שמגיעים לגורם-ניהולי/תרבותי.

### 5 צעדים

1. **זיהוי האירוע** שגרם לתאונה + זיהוי האירוע שגרם לפגיעה (נפרדים!)
2. **לכל אירוע**, שואלים: **למה?** מה גרם לו? (תיאור קצר: שם-עצם + פועל)
3. **חוזרים** על השאלה לפחות **3 פעמים-או-יותר**
4. **מפסיקים** כאשר מגיעים ל-**גורם-שורש** (בד"כ ניהול-בטיחות / תרבות-בטיחות)
5. ניתן להציג את הניתוח באופן **גרפי** (עץ של "למה")

### דוגמת-תרגיל (מהמצגת) — פטיש נפל מפיגום

תאונה-עיקרית: **שבר בגולגולת של פועל בחצר-האתר**

ענף "פטיש-נפל":

- **למה?** פועל בעט בפטיש → לא שם-לב שיש פטיש (אחרי 9 שעות-עבודה ⚠️)
- **למה?** פטיש מונח ליד קצה רצפת-הפיגום → אין חגור לכלי-עבודה
- **למה?** לא היה לוח-רגל בפיגום → פיגום לא נבנה ע"י בונה-פיגומים מוסמך
  - **שורש**: לא בוצע סקר-סיכונים / PTP (Pre-Task Planning)

ענף "שבר-בגולגולת":

- **למה?** לא חבש קסדת-מגן → אין אכיפה
- **למה?** עבד מתחת לפיגום → לא פינו אזור-הסיכון
  - **שורש**: לא בוצע סקר-סיכונים / PTP

**מסקנה**: 2 גורמי-שורש שונים → 2 המלצות נפרדות (אכיפת-PPE + ביצוע-PTP).

## 8.2 — מודל RCA (Root Cause Analysis) — דגם גורמים-שורשיים

### מהות

**RCA** = שיטה היררכית של 4 רמות-גורמים. מסדרים את הגורמים מהרמה-החיצונית-ביותר (רגולציה) לרמה-הפנימית-ביותר (ביצוע).

### 4 רמות-גורמים (מבחוץ-פנימה)

```
[רגולציה] ──→ [גורמי ניהול הבטיחות (ניתוח-סיכונים)]
                            ↓
[גורמים חיצוניים] ──→ [גורמים מתווכים (נהלים, עיצוב-מטלה, פיקוח, הכשרה)]
  (סביבה וחבלה)                  ↓
                       [גורמי ביצוע (אדם וציוד)]
```

**יישום**:

- חקירת-תאונה מתחילה ב-**גורמי-ביצוע** (מה קרה בפועל)
- עוברת ל-**גורמים-מתווכים** (מה אפשר את הביצוע)
- וממשיכה ל-**גורמי-ניהול** (למה הנהלים היו חסרים)
- בסופו של דבר — נוגעת ב-**רגולציה** (האם יש פערים בחוק)

## 8.3 — Barrier Analysis (ניתוח-מחסומים)

### מהות

**מחסום** = כל מה שמונע מכשל לגרום ל-**אובדן-שליטה** (loss of control). ניתוח-מחסומים בוחן: אילו מחסומים היו? אילו כשלו? אילו חסרו?

### דוגמאות-מחסומים

| חסר/כשל              | מחסום-נדרש                      |
| -------------------- | ------------------------------- |
| חוסר ידע של העובד    | **מידע + הדרכה**                |
| פגעי מזג-אוויר       | **היערכות-מראש / דחיית-פעילות** |
| תחזוקה לקויה         | **נהלים**                       |
| מכונה לא-מוגנת       | **גידור**                       |
| פליטת-גז ממערכת גפ"מ | **בדיקה תקופתית**               |

### דוגמה-היסטורית: אסון **FLIXBOROUGH** (1.6.1974, UK)

פיצוץ בית-זיקוק בבריטניה. הוכחה-קלאסית לכישלון Barrier Analysis: כמה מחסומים-עקפו.

### מודל-המחסומים — 2 רמות

**מחסומים פיזיים** (3 שכבות + סילוק):

1. **סילוק מקור-האנרגיה** (אופציה אידיאלית)
2. **מחסום על מקור-האנרגיה** ("גידור לבטח") — בידוד-מקור
3. **מחסום בנתיב-מעבר/הסטה** — חציצה-בין-מקור-לאדם
4. **מחסום-המגן על אדם** (ציוד-מגן-אישי)

**מחסומים מנהלתיים** (5 שכבות):

1. **התרעות** (alerts, alarms)
2. **עיצוב-מטלה / משמרות** (work design)
3. **נהלים והוראות**
4. **מיון וגיוס-עובדים** (right person for the job)
5. **הדרכה**

### חשוב להבדיל

- **תאונה** = **אובדן-שליטה** על אנרגיה (גם אם לא נפגע אדם)
- **פגיעה** = תוצאה של האנרגיה הפנויה הפוגעת באדם

לכן מחסומים לא רק מונעים-פגיעה אלא גם מונעים-אובדן-שליטה.

## 8.4 — תיאוריות הדומינו (Domino Theories)

### תיאוריית הדומינו של היינריך (Heinrich, 1941)

**5 פרוסות-דומינו** שכל-אחת מפילה את הבאה:

```
[השפעות תורשתיות + סביבתיות]
        ↓
[אשמת אדם]
        ↓
[פעולה לא-בטוחה / מצב לא-בטוח]
        ↓
[תאונה]
        ↓
[פגיעה]
```

**3 תובנות-עיקריות מהמודל**:

1. הגורמים-המיידיים לתאונה = **פעולה לא-בטוחה + מצב לא-בטוח**
2. תאונה = **תוצאה של שרשרת-אירועים** שקדמו לאירוע (לא מזל / גורל!)
3. **סילוק אחד-הגורמים** שקדמו לתאונה ימנע את אירוע-התאונה

**ביקורת**: שמה דגש מוגזם על "אשמת-אדם" — לא לוקחת בחשבון את ההנהלה.

### תיאוריית הדומינו של בירד ולופטוס (Bird & Loftus, 1976)

**5 פרוסות-דומינו עדכניות**:

```
[הנהלה: חוסר שליטה]
        ↓
[סיבות בסיסיות: גורמי אנוש + עיצוב מטלה]
        ↓
[סיבות מיידיות: פעולה לא-בטוחה + מצב לא-בטוח]
        ↓
[אירוע: מגע עם מקור אנרגיה]
        ↓
[אובדן: אנשים + ציוד]
```

**התרומה החדשה**: הקביעה-התיאורטית ש-**ההנהלה היא אחראית-העל לאירוע תאונת-עבודה** (לא העובד שנפגע).

**מסקנה לפעולה**: מניעת-תאונות תבוצע בעיקר ע"י **שיפור שליטת-ההנהלה** (תכנון, ארגון, פיקוח, אספקת משאבים), ולא ע"י איתור עובדים-אשמים או כשלים-מכניים.

## 3.8.1 — טוקסיקולוגיה (Toxicology)

### עקרון Paracelsus (1493-1541)

> "**Sola dosis facit venenum**" — _רק המינון עושה את הרעל_

מאז Paracelsus: "**כל החומרים הם רעילים** — אין חומרים שאינם-רעילים. רק המינון עושה את הרעל".

### **LD50** — Lethal Dose 50% (קריטי!)

- **מינון-קטלני קטלני 50%** = מינון הדרוש להרוג 50% מאוכלוסיית-הנבדקים
- מומחת ב-1927 ע"י **J.W. Trevan**
- **תחת MeSH** = "הכמות של רעלן, חומר טוקסי או עוצמת-קרינה הדרושים על-מנת להרוג 50% מאוכלוסיית-הנבדקים"

### דירוג-רעילות לפי LD50

| LD50             | קטגוריה        | דוגמה                                |
| ---------------- | -------------- | ------------------------------------ |
| **<1 mg/kg**     | רעיל-ביותר     | קלפסון (Clostridium botulinum toxin) |
| **1-50 mg/kg**   | רעיל           | פוטה, נחשים                          |
| **50-500 mg/kg** | רעילות-בינונית | חומ"ס תעשייתיים                      |
| **>500 mg/kg**   | לא-רעיל בינוני | מלח-בישול: LD50 = 3,000 mg/kg        |

### NOEL — No-Observed-Effect-Level

הריכוז-הגבוה-ביותר בו לא-נראית-תגובה בנבדקים.

### תופעות-אינדיבידואליות

- **היפר-סוסקוסטיביות**: יחידים רגישים-יותר מהאוכלוסייה הכללית
- **היפו-סוסקוסטיביות**: יחידים פחות-רגישים

## 3.7 — קרינת-לייזר (הרחבה)

> **מקור**: סעיף ב-PDF רפואה תעסוקתית — "תקנות הבטיחות (גיהות תעסוקתית ובריאות העוסקים בקרינת-לייזר), התשס"ה-2005"

### תקנה — קצרה ותמציתית

**תקנה-יחידה לעבודה עם לייזר**: תקנות הבטיחות בעבודה (גיהות-תעסוקתית ובריאות-העוסקים בקרינת-לייזר), **תשס"ה-2005**.

### LASER — הגדרה

**LASER** = **L**ight **A**mplification by **S**timulated **E**mission of **R**adiation = הגברת-אור על-ידי הקרנת-פליטה-מאולצת.

### חובות-מעסיק (לפי התקנה)

1. **ממונה על בטיחות לייזר** — בכל מקום-עבודה שבו עוסקים בלייזר. חייב:
   - לבדוק עמידה בחוקים
   - להעלאת-מודעות בנושאי-בטיחות, גיהות
   - הדרכת-עובדים על-הסיכונים
2. **קורס-בטיחות-לייזר** — מי שמשמש כממונה חייב לעבור קורס מבוצע על-ידי מכון-מאושר. הקורס דורש להדריך, וועומד בכל הדרישות בתקנות ובנוהל.

### מקורות-קרינה ופגיעות-עיניים

מקורות UV: ריתוך, מנורות פלורסנט, מנורות חיטוי-חיידקים, **שמש**.

קרינת-לייזר חודרת:

- **UVC** (200-280nm) — בלוטות-לחות הקרנית
- **UVB** (280-315nm) — קרנית + עדשה
- **UVA** (315-400nm) — רטינה
- אור-נראה (400-760nm) — רטינה ומקולה

## 3.5.1 — תקנות בנזן 1983 (נוסח רשמי) ⭐

> **מקור**: PDF "בנזן 1983" (אולג גולובין) — פורסם תשמ"ד-1983; עדכונים תשנ"ט-1999, תשע"א-2011

### הגדרת "בנזן (בנזול)" — קריטי!

**בנזן** = חומר שנוסחתו הכימית **C₆H₆**, **לרבות תכשיר המכיל יותר מ-1% בנזן** מנפח-המסיסה (תק' תשע"א-2011).

### 🔴 איסור-השימוש בבנזן (סעיף 2 — קריטי!)

**לא ישתמשו בבנזן** במפעל או במקום-עבודה אלא ל-**4 מטרות-מותרות**:

| #            | מטרה-מותרת                                                     |
| ------------ | -------------------------------------------------------------- |
| 1            | ייצור / עיבוד / טלטול של **בנזן** עצמו                         |
| 2            | ייצור / עיבוד / טלטול של **דלק למנועים**                       |
| 3            | עבודה ב-**סינתזה כימית** שבה הבנזן מרכיב-הכרחי                 |
| 4 (תק' 1999) | עבודה **אנליטית או מחקרית במעבדה** — דורש **היתר ממפקח-אזורי** |

**שאר כל-שימוש בבנזן — אסור!** (בעבר היה משמש כמסיס נפוץ, היום אסור עקב סרטן-הדם — Leukemia)

### חישוב-חשיפה משוקלל (סעיף 3)

> **חשיפה משוקללת = (C₁·T₁ + C₂·T₂ + … + Cₙ·Tₙ) / סה"כ-משך-הדגימה**

- **C** = ריכוז-בנזן באוויר, חלקיקים-למיליון (ppm)
- **T** = משך-הדגימה, בשניות/דקות

### בדיקות-סביבתיות (סעיף 4) — קריטי!

| פרמטר        | ערך                                                   |
| ------------ | ----------------------------------------------------- |
| **תדירות**   | אחת ל-**3 חודשים-לפחות**                              |
| מבצע         | בודק-מעבדתי-מוסמך, ע"י מעבדה-מוסמכת                   |
| שמירה במפעל  | **20 שנים** (פנקס-בריאות-העובד)                       |
| שמירה במעבדה | **50 שנים**                                           |
| ידיעה        | למפקח-אזורי + שירות-רפואי-מוסמך + פרסום בתחנות-עובדים |

### 6 אמצעי-גהות-סביבתיים (סעיף 5)

1. אמצעי-יניקה, ניקוי, פליטה
2. **מנדפים-יעילים** בעבודה בשקילה/בדיקות
3. איסוף + סילוק-מיידי של פסולת
4. **מקלחות-חירום** במספר-מספיק (גוף + עיניים)
5. חדרים-מיוחדים לאכילה / שתייה / מנוחה
6. **בגדי-עבודה** + נעליים + גרבים + משקפי-מגן + משחת-מגן-לידיים

### דגשים-קריטיים לוועדה ⚠️

- בנזן הוא **מסרטן ידוע** — Leukemia
- ה-**1% threshold** קובע אם תכשיר נחשב "בנזן"
- **3 חודשים** = תדירות-בדיקה (לא חצי-שנה!)
- **20+50 שנים** שמירה (קריטי לתביעות-עתידיות)
- **רמת-הפעולה (AL)** = מחצית-ה-TWA (עיקרון-חוזר לכל-חומר)
