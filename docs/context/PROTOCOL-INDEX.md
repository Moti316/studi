## 1. שכבות-הממשל — מי מדווח למי

מערכת-הסוכנים בנויה מ-**4 שכבות-ביצוע** (מועצה → מתווך → ראשי-צוות → מבצעים) ו-**ענף-בקרה עצמאי** המקביל למתווך ומדווח ישירות למועצה.

```
                          מועצה (מוטי) — Council  ← סמכות-על · אישור-סופי · Star-Chamber
                                  ▲
              ┌───────────────────┴────────────────────────┐
        מתווך (אמיר) — mediator              ענף-בקרה (oversight) — עצמאי
        ניתוב · קונפליקט · בקרת-סחף           מדווח ישירות למועצה (לא דרך המתווך)
        aggregate-report → מועצה              oversight-lead (נדב) · צו-עצירה קוורום-2/3
              ▲                                          ▲
   ┌──────────┼──────────┬──────────────┐      ┌─────────┴──────────┐
strategic  builder    quality   coordinator   בקרה-חיצונית      מבקר-תכנית
 -lead      -lead      -lead      -lead         עידו · הדס        רותם · שני · גיא
 (אבירם)   (יונתן)    (מירב)     (דורון)
   ▲          ▲          ▲          ▲
   3   →     13   →      5   →      1     =  22 מבצעים           6 מבקרים (ענף-בקרה)
```

**רוסטר: 33 סוכנים** = 22 מבצעים + 4 ראשי-צוות + 1 מתווך + 6 ענף-בקרה. הרוסטר המלא: [`../../teams/README.md`](../../teams/README.md).

| שכבה                          | מי                                    | אחריות-ליבה                                                        | מדווח ל                       |
| ----------------------------- | ------------------------------------- | ------------------------------------------------------------------ | ----------------------------- |
| **מועצה** (Council)           | מוטי                                  | חזון · אישור-סופי · שובר-תיקו עליון · Star-Chamber                 | — (סמכות-על)                  |
| **מתווך** (mediator)          | אמיר (`opus`)                         | ניתוב חוצה-צוותים · הכרעת-קונפליקט · בקרת-סחף · `aggregate-report` | מועצה                         |
| **ראש-צוות** (`*-lead` ×4)    | אבירם · יונתן · מירב · דורון (`opus`) | תיאום-צוות · קונסולידציה ל-`control-report` · שער-איכות            | מתווך                         |
| **מבצע** (×22)                | מומחי-תחום (רובם `sonnet`)            | ביצוע בתוך Scope+Red-Lines · תיעוד-עצמי · דיווח                    | ראש-הצוות                     |
| **ענף-בקרה** (`oversight` ×6) | נדב + 5 (`opus`/`sonnet`)             | ביקורת בלתי-תלויה (read-only) · צו-עצירה · `oversight-report`      | **מועצה ישירות** (עוקף-מתווך) |

> **עיקרון-זרימה:** סוכן-ביצוע לעולם אינו מדבר ישירות עם המועצה — `סוכן → ראש-צוות → מתווך → מועצה`, וההכרעות חוזרות באותו ערוץ הפוך. **שני חריגים בלבד:** (1) הפרת-red-line קריטית → דגל-חירום שעוקף את ראש-הצוות ישר למתווך. (2) ענף-הבקרה → מדווח ישירות למועצה by-design (כי הוא מבקר את המתווך עצמו).

## 3. זרימת-הדיווח · בקרת-סחף · צו-עצירה

### קצב-דיווח (Reporting Cadence)

מסלול-הדיווח עולה שכבה-אחר-שכבה: **control-report (צוות) → aggregate (מתווך) → מועצה**. אין דילוג-שכבה כלפי-מעלה, פרט לדגל-חירום ולערוץ-הבקרה-הישיר.

| תדירות         | מי → מי                  | ערוץ                                | תוכן                                         |
| -------------- | ------------------------ | ----------------------------------- | -------------------------------------------- |
| כל משימה       | סוכן → `activity-log.md` | קובץ-עצמי                           | 6 שדות התיעוד-העצמי                          |
| כל משימה       | סוכן → ראש-צוות          | `control-report.md`                 | Outcome + Verdict + דגלים                    |
| סוף-מחזור      | ראש-צוות → מתווך         | `aggregate-report.md`               | קונסולידציית-צוות + סטטוס-דגלים              |
| סוף-מחזור / סף | מתווך → מועצה            | `aggregate-report.md`               | מצב-כללי + הכרעות-נדרשות + הסלמות            |
| לפי-אירוע      | סוכן → מתווך (דגל)       | `comms/` BROADCAST                  | הפרת-red-line / סחף קריטי (עוקף ראש-צוות)    |
| סוף-מחזור / סף | ענף-בקרה → מועצה         | `oversight-report.md`               | תמונת-בקרה בלתי-תלויה (**ישיר, עוקף-מתווך**) |
| לפי-אירוע      | ענף-בקרה → מועצה         | `stop-orders-ledger.md` + BROADCAST | צו-עצירה קוורום-2/3 + ראיה (רק-מוטי-מבטל)    |

### בקרת-סחף (Drift Control)

**סחף** = סוכן שפועל מחוץ ל-`identity.md` שלו (חריגת-Scope · התעלמות-מ-red-line · תוכן-מומצא · שגיאה-חוזרת). **3 חיישנים:** (1) Self-check נכשל ב-`activity-log` · (2) שער-צוות מזהה אי-התאמה · (3) loop-breaker (אותה error-hash ×3 → `CRITICAL STOP`).

**סולם-הסלמה:** ראש-צוות מסמן דגל ב-`control-report` → המתווך מכריע **שיקום** (תיקון-במקום + הבהרת-identity) מול **החלפה** (ארכוב ל-`_archive/<slug>-<date>/` + `REASON.md` + סוכן-חדש עם `memory.md` נקי). הפרת-red-line קריטית (secret שדלף / תוכן-מומצא שפורסם) → המתווך עוצר מיידית ומעלה למועצה. הפירוט: [`../../teams/ORG.md`](../../teams/ORG.md) §בקרת-סחף.

### צו-עצירה (Stop-Order) — סמכות ענף-הבקרה

הסמכות החתומה של ענף-הבקרה. **מנפיקים** כש: סטייה-מאומתת מהתוכנית · הפרת-red-line (תוכן-מומצא · secret · תלות-NotebookLM-בפרודקשן · הפרת-firewall-מגן · ידע-קורס-ב-`src/`) · סוכן-סורר.

- **קוורום 2/3** — שניים משלושת חברי-תת-הצוות הרלוונטי חותמים (בקרה-חיצונית: נדב/עידו/הדס · מבקר-תכנית: רותם/שני/גיא).
- **סיבה + ראיה** — קובץ/שורה/commit.
- **חותמת ב-[`stop-orders-ledger.md`](../../teams/oversight/stop-orders-ledger.md)** + **`<<BROADCAST>>`** ב-`comms/`.
- **אפקט:** עוצר את **המסלול-הספציפי** (לא את כל-המערכת) **עד אישור-מועצה**. **רק מוטי מבטל** — לא ראש-הענף, לא הקוורום.

> סטטוס נוכחי: אין צווי-עצירה פעילים (הענף הוקם 2026-06-03).

## 4. מחזור-חיים של סוכן · 12-שדות-identity · פרוטוקול 7-הצעדים

### פרוטוקול-עבודה — 7 צעדים (כל סוכן, כל משימה)

1. **קליטה** — צעד-0: `git fetch`/`pull` (multi-machine). אז: `PROJECT-CONTEXT` → `identity` → `memory` → תדריך-המשימה.
2. **גבולות** — פועל רק בתוך Scope+Red-Lines; מחוץ-לתחום → הסלמה (`ראש-צוות → מתווך → מועצה`), לא ניחוש.
3. **ביצוע** — TDD-first · מול הסכמה-שבפועל (לא ADR-בלבד) · RTL/a11y · אפס-secrets.
4. **תיעוד-עצמי** — רשומה ל-`activity-log.md` (6 שדות: Outcome · What-changed · Verification · Follow-ups · Verdict · Self-check).
5. **למידה** — מעשיר "לקחים-מצטברים" ב-`memory.md`.
6. **דיווח** — `סוכן → ראש-צוות (control-report) → מתווך (aggregate) → מועצה`.
7. **בקרת-סחף** — סטייה/הפרת-red-line → דגל מיידי (עוקף ראש-צוות → מתווך).

### 12-שדות ה-identity (שכבה A: 1-11 ניטרלי · שכבה B+C: 12 פרויקטי)

1. Mandate · 2. Professional Standard · 3. Methodology & Sources · 4. Decision Framework · 5. Scope Boundaries · 6. Red Lines — never do · 7. Interfaces & Handoffs · 8. Escalation Path · 9. Output Contract · 10. Definition of Done / KPIs · 11. Anti-patterns · 12. Project Focus _(השדה היחיד התלוי-פרויקט — מתרענן ב-pivot)_.

### מחזור-חיים (Lifecycle)

- **הקמה:** 6 צעדים ([`HOWTO-add-agent.md`](../../teams/HOWTO-add-agent.md)) — slug+tier+model · תיקייה עם 3 קבצים · stub ב-`.claude/agents/` · רישום ברוסטר · רישום ב-CLAUDE.md+PROJECT-MAP · prettier+commit.
- **מחיקה/החלפה (כשהמתווך מכריע "החלפה"):** הסרה-מהרוסטר → ארכוב **כפי-שהוא** ל-`_archive/<slug>-<date>/` + `REASON.md` → סוכן-חדש עם `memory.md` **נקי** (לא יורש זיכרון-סורר). **לעולם לא מוחקים זיכרון לתמיד — מארכבים.**

## 5. פרוטוקול-החלטות — Star-Chamber (רב-מודלי)

הכרעות-מפתח במועצה אינן נסמכות על מודל-יחיד (הזיות · ביטחון-יתר). הפרוטוקול: [`../../teams/strategic/_debate-protocol.md`](../../teams/strategic/_debate-protocol.md).

**3 שלבים:** (1) **איסוף-הקשר** ל-prompt ניטרלי נטול-רמזים. (2) **fan-out מקבילי** — אותה בקשה למספר מודלים בבידוד. (3) **אגרגציה** ל-JSON קשיח: **קונצנזוס** (כולם → אישור-מיידי) · **רוב** (2/3 → בירור) · **תצפית-יחיד** (השערה) · **אי-הסכמה → מצב-פולמוס**.

**מצב-פולמוס** = כללי Chatham House: קילוף-זהות-המודל מהטיעונים → סיכום-אנונימי לבחינה-חוזרת → "שופט" שוקל-ראיות. **מיתון 4 הטיות:** מיעוט-צודק (ראיות > סטטיסטיקה) · התקרנפות-מתשישות (תקרת 3-4 סבבים) · זיוף-זהות (קילוף-תגים) · פברוק-ביקורת (לנסח "מה חסר?" לא "חפש שגיאות").

> אישור-סופי תמיד = מוטי (המועצה). ה-Star-Chamber הוא כלי-החלטה, לא תחליף-סמכות.
