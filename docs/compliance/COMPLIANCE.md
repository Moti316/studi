# COMPLIANCE — חובות-ציות, אבטחה, פרטיות ונגישות

> **מסמך-עוגן לנושא הציות של StudiBuilder.** נכתב 2026-05-31 · בעלים: צוות-הציות (ראה למטה).
> ⚖️ **הסתייגות:** מסמך-עבודה פנימי — **לא ייעוץ משפטי.** לפני מכירה מסחרית, **בדיקה משפטית מקצועית** (פרטיות + צרכנות + נגישות) היא חובה.

## עיקרון-יסוד

החובות חלות על **כל השירות לציבור** (התחברות · דשבורד · שיעורים · checkout) — **לא רק על דף-המכירה**:

- **פרטיות + אבטחה** — חלות **כבר עכשיו** (אנו אוספים מידע אישי: מיילים, נתוני-לימוד).
- **נגישות** — חלה על **כל האתר הציבורי**.
- **הגנת-צרכן + תשלומים** — מתגבשות **ברגע שמתחילים למכור בתשלום**.

> בניית הציות **מהיסוד (עכשיו)** זולה בהרבה מאשר בדיעבד. כבר היום אנו בונים RTL+a11y, security headers ו-auth מאובטח.

## כלל-זכויות (זכויות-יוצרים) — חקיקה + חומרי-מרצה

> מקור-אמת לכלל-הזכויות בתוכן-הקורסים. כל פלט **name-clean**. ראה `../../courses/safety-officer/ATTRIBUTION.md` + זיכרון `no-lecturer-names-copyright`.

- **חקיקה (חוקים/תקנות/פקודות) = נחלת-הכלל** → שימוש-ישיר מותר (ציטוט + מקור מאומת); בסיס-האימות של הקורס.
- **תקני ISO / ת"י = בתשלום ומוגנים** → **תקצירים-מקוריים בלבד**; אסור לשכפל/לצטט את טקסט-התקן.
- **חומרי-מרצה** (מצגות/סיכומים/הקלטות/בנקי-שאלות ממותגים) = **reference + שכתוב-מהותי**; **ללא שמות-מרצים/תלמידים/מנחים בשום פלט**.
- **חומרים ממותגים** (כולל עבודות-גמר לדוגמה) = הכרעה **מקרה-במקרה עם מוטי**.

## צוות-הציות (task-force מסוכנים קיימים)

לפי פרוטוקול ה-Agent-OS (`teams/HOWTO-add-agent.md`) — משתמשים בסוכנים קיימים, לא מקימים חדשים בלי פער-אמיתי. הוקם **כוח-משימה חוצה-צוותים**:

| תחום                                      | סוכן                         | תפקיד בכוח-המשימה    |
| ----------------------------------------- | ---------------------------- | -------------------- |
| פרטיות / הגנת-מידע                        | **privacy-officer** (עדן)    | **מוביל כוח-המשימה** |
| אבטחה / תקנות-אבטחה / סליקה               | **appsec** (שחר)             | בקרת-אבטחה           |
| נגישות (5568/WCAG AA)                     | **accessibility-i18n** (גיל) | עמידה + הצהרת-נגישות |
| ניסוח-משפטי (policy/terms/a11y-statement) | **content-writer** (שירה)    | כתיבת-מסמכים         |
| הגנת-צרכן + מוכנות-לאנץ'                  | **product-owner** (נועה)     | זרימת-רכישה/ביטול    |
| דיוק חקיקתי-domain                        | **domain-expert** (רון)      | ייעוץ                |
| תיאום ודיווח למועצה                       | **mediator** (אמיר)          | אגרגציה → מוטי       |

> **אופציה עתידית:** אם הנושא גדל — להקים סוכן ייעודי `compliance-officer` (tier `quality`) לפי 6 הצעדים ב-HOWTO. כרגע ה-task-force מספיק.

## צ'קליסט-ציות (סטטוס)

**✅ כבר קיים:** RTL + a11y מהיסוד (ARIA/מקלדת/ניגודיות/reduced-motion) · security headers (next.config) · auth מאובטח · secrets ב-`.env.local` בלבד.

**⬜ נגישות:** עמידת-AA מאומתת · הצהרת-נגישות · בדיקת-נגישות אוטומטית (axe) ב-CI · FAB-נגישות (כבר במפרט).
**⬜ פרטיות:** מדיניות-פרטיות · באנר/הסכמה · זכויות-נושא-מידע (עיון/מחיקה) · נוהל אירוע-אבטחה · בידוד `SUPABASE_SERVICE_ROLE_KEY` (server-only).
**⬜ צרכנות:** גילוי-מלא בעמוד-מכירה · זרימת ביטול-עסקה (14 יום) · תנאי-שימוש.
**⬜ תשלומים:** ספק-סליקה · חשבונית/מע"מ.
