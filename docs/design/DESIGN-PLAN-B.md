# DESIGN-PLAN-B — StudiBuilder · כיוון "Premium-Clean · מקצועי-אמין"

> **אתוס:** נקי · רגוע · רציני-ומכובד. מינימליזם **חם** (לא סטרילי). תחושת אמון ואיכות-פרימיום, מותאם לקהל-מקצועי בוגר (ממוני-בטיחות).
> **מטרה:** מוצר-לשיווק → איכות-עיצוב גבוהה, מראה נקי/נעים/מזמין. כיוון B מתוך 3.
> **גרסה:** 1.0 · **תאריך:** 2026-06-04 · **שפה:** עברית RTL כאזרח-ראשון.
> 🔒 **FIREWALL:** StudiesGo = מקור-ההשראה החזותי **היחיד**. שאלון-ה-onboarding נכתב native מהשראת-מבנה בלבד (`docs/IDEAS.md` "מתכנן-מסלול-אישי"). **אפס-העתקה** מקוד/תוכן/prompts של מערכת "מגן" (megen).

## 0. תמצית-הכיוון (one-liner לכל בעל-עניין)

> **"שקט-תכליתי שמרגיש יקר."** פחות צבע, יותר רווח-לבן. כחול-עמוק-מאופק כעוגן-מותג, כתום-ענבר כ**מבטא-נקודתי בלבד** (לא דומיננטי). טיפוגרפיה גדולה-ונושמת, כרטיסים שטוחים עם hairline-borders וצללים רכים-מאוד. תנועה עדינה ומהוגנת (פחות overshoot, יותר ease). המסקוט "רוני" נוכח אך מאופק — מדריך-עמית בוגר, לא מסיח. **התוצאה:** מוצר שממונה-בטיחות בן-50 ירגיש שמכבד אותו, ושנראה כמו מנוי-פרימיום ששווה לשלם עליו.

**3 מילות-המפתח שמנחות כל החלטה:** `נקי` · `אמין` · `יקר-ברוגע`.

**מתח-מכוון (tension) שהכיוון פותר:** StudiesGo הוא צעיר-ואנרגטי (ניגוד כחול↔כתום חזק, spring-overshoot בכל-מקום, gamification בולט). הקהל שלנו בוגר-ומקצועי. כיוון-B **משמר את חתימת-StudiesGo** (ניגוד כחול↔כתום · pill-buttons · flat-light-cards · glass-dark-nav · mascot-מרחף · spring) אך **מנמיך את הווליום**: הכתום הופך ממבטא-שווה-ערך ל-accent-נקודתי, ה-spring מאבד overshoot, וה-gamification עובר לטון מהוגן. כך נשמר ה-DNA אך מושג טון-פרימיום.

- **משפחה:** `Rubik` (primary) → `Assistant` (fallback) → `system-ui, sans-serif`. שניהם variable-Hebrew מצוינים ל-RTL. כיוון-B מעדיף **משקלים מתונים** (פחות 700 בכל-מקום) לטון-מהוגן.

### 1.5 צללים (elevation)

**עיקרון-B:** light = **כמעט-שטוח** (חתימת-StudiesGo) אך עם **צל רך-מאוד ורחב** (לא רק border) שנותן עומק-פרימיום עדין. dark = glow-rings + float.

```css
/* Light */
--shadow-hairline: 0 0 0 1px var(--border-default); /* כרטיס-בסיס: border בלבד */
--shadow-soft:
  0 2px 8px -2px rgba(26, 34, 51, 0.06), 0 1px 2px rgba(26, 34, 51, 0.04); /* כרטיס-מרומם עדין */
--shadow-card:
  0 4px 16px -4px rgba(26, 34, 51, 0.08), 0 2px 4px rgba(26, 34, 51, 0.04); /* modal/dialog */
--shadow-float: 0 12px 32px -8px rgba(27, 79, 214, 0.14); /* CTA-ראשי hover (גוון-מותג) */
--shadow-focus: 0 0 0 3px rgba(27, 79, 214, 0.22); /* focus-ring (a11y) */

/* Dark */
--d-shadow-float: 0 10px 25px -5px rgba(0, 0, 0, 0.5);
--d-glow-primary: 0 0 0 1px rgba(59, 111, 224, 0.4), 0 0 24px rgba(59, 111, 224, 0.25);
--d-glow-cyan: 0 0 20px rgba(34, 184, 214, 0.3);
--d-glow-accent: 0 0 18px rgba(245, 166, 35, 0.28);
```

> **הבדל-B מ-StudiesGo:** StudiesGo light = flat-כמעט-ללא-shadow (border בלבד). כיוון-B מוסיף `--shadow-soft` עדין-מאוד למשטחים-אינטראקטיביים — מספיק כדי לתת עומק-פרימיום, מעט מספיק כדי לשמור על נקיון. כרטיסים-סטטיים נשארים על `--shadow-hairline`.

### 1.6 אייקונוגרפיה

- **משפחה:** `Lucide` (stroke-based, `1.75px` stroke — מעט-עבה-יותר מ-default ל-presence-פרימיום), 24px-grid, מעוגל-קצוות (`stroke-linecap: round`).
- **טון-B:** outline-בלבד (לא filled) ברוב-המקרים — נקי-ומהוגן. filled שמור ל**מצב-פעיל** (tab-active, toggle-on).
- **אייקוני-נושא (קורס-בטיחות):** קסדה, מגן, מסמך-מאושר, התראה-משולש — תמיד outline, צבע `text-secondary` במצב-רגיל, `primary` במצב-פעיל.
- **אייקוני-בחירה (onboarding theme-cards):** איקון-צבעוני-בעיגול (חתימת-StudiesGo) — עיגול ב-`primary-soft`/`accent-soft`, איקון בצבע-מלא. נשמר.
- **גודל-קנוני:** 16 (inline) · 20 (כפתור) · 24 (nav, header) · 32-40 (empty-state, theme-cards).

### 1.7 רכיבי-ליבה (טוקנים יישומיים)

- **Header-quiz:** סרגל-התקדמות-נקודתי (נקודות `border-default`, נקודה-נוכחית מתמלאת `accent`) · מד-XP (`30 XP` + איקון-ברק `accent`) · streak (`🔥 3` → גרסת-B: איקון-flame `accent`, מספר `text-primary`) · כפתור-ניווט-עגול `surface-card`+`shadow-soft` · `AiNotice` קבוע (ראה 7.3).
- **Header-dashboard:** 3 counters בכרטיסי-pill (credits · XP · streak) — בכיוון-B הם `surface-card` + `shadow-hairline`, מספר `text-primary` 600, איקון-accent. ברכה-תלוית-שעה ב-`h1` עם שם-המשתמש ב-`primary`.
- **Bottom-nav:** 4 tabs (בית · קורסים · סטטיסטיקות · הגדרות). tab-פעיל = כרטיס `primary-soft` מרומם + איקון-filled-`primary` + label-`primary`-600. שאר = outline-`text-muted`.
- **ViewToggle:** זוג-pills בתוך מסילת-`surface-sunken`. פעיל = `surface-card`+`shadow-soft`+טקסט-`primary`. (גרסת-B מאופקת — לא "כחול מלא vs כתום מלא" אלא pill-לבן-מורם על מסילה).
- **StreakCard:** טבעת-flame `accent` + לוח-7-ימים (היום = `primary`-מלא, ימים-שהושלמו = `accent`-מלא, עתידיים = `border-default`).
- **DailyProgressCard:** זוג-כרטיסים (XP `0/20` · שיעורים `0/1`) — progress-ring `accent`.
- **Selector-buttons:** יעד-יומי (4 pills, נבחר = `accent`-border + `accent-soft`-bg) · ערכת-נושא (3 כרטיסים גדולים, איקון-בעיגול-צבעוני).
- **Toggle/Switch:** דלוק = `primary`-מלא, knob-לבן + `shadow-soft`; כבוי = `border-strong`.
- **Input + tip-card:** שדה `surface-sunken` rounded-`r-input` + `shadow-focus` כחול ב-focus + כרטיס-"💡 טיפ קטן" `bg-app-warm` מתחתיו (חתימת-StudiesGo, גוון-חם בכיוון-B).
- **Badge "מקור" (citation-affordance — חדש-לכיוון):** chip-קטן `primary-soft` + איקון-מסמך + טקסט "מקור: תקנה X §Y, עמ׳ Z" — מחזק-אמון (ראה 9).

## 2. מסך התחברות (Auth / Login)

### 2.1 קונספט

modal-glass מרכזי על רקע-gradient עדין (`bg-app` → `primary-soft` עדין-מאוד באלכסון-RTL), מרכז-מסך, רוחב מקסימלי `400px`. מסקוט "רוני" (pose: curious) מציץ מעל-הכרטיס בפינה — מקדם-בברכה אך לא-ילדותי. הטון: **שער-כניסה מכובד למוצר-פרימיום**, לא טופס-טכני.

**עקרון-B:** מינימום-שדות. ברירת-המחדל הראשונה = **Google OAuth** (כפתור-ראשי, חיכוך-אפס), magic-link כאלטרנטיבה-משנית. אין סיסמאות בכלל.

### 2.2 פריסה ורכיבים

- **Glass-card** (`surface-card` 95% + `backdrop-blur(8px)`, `r-modal`, `shadow-card`).
- **לוגו StudiBuilder** (קטן, למעלה-מרכז) + tagline "בניית-קורסים שמכבדת את הזמן שלך".
- **Mascot "רוני"** curious, מרחף (idle-float), קסדת-בטיחות כתומה — חתימת-הקורס.
- **GoogleSignInButton** — כפתור-ראשי full-width, pill, `surface-card`+`border-strong`+לוגו-Google (סטנדרט-מותג Google, לא-מצבע).
- **מפריד** "או" (קו-hairline + טקסט-`text-muted`).
- **MagicLinkForm** — שדה-email (`surface-sunken`, focus-ring) + כפתור-`primary` "שלח קישור".
- **Footer-קטן:** "בכניסה את/ה מאשר/ת תנאי-שימוש ומדיניות-פרטיות" (קישורים `primary`, caption).

### 2.3 מצבים (states)

## 3. מסך פתיחה + שאלון-משתמש (Onboarding)

> 🔒 native לחלוטין. השראת-מבנה בלבד מ-`docs/IDEAS.md` "מתכנן-מסלול-אישי". אפס-שימוש ב-`study_plan_90days`/`committee_bank`/תוכן-מגן. הנוסחה והשאלות נכתבו כאן ל-StudiBuilder.

### 3.1 זרימת-מסכים (flow)

```
[Welcome] → [Q1 תאריך] → [Q2 זמן-יומי] → [Q3 רקע] → [Q4 התקדמות] → [Q5 ימים] → [סיכום: המסלול-שלי] → /dashboard
```

- כל-שאלה = **מסך-יחיד** (פוקוס, ללא-עומס). מעבר slide-אופקי-RTL בין-שלבים.
- **מד-התקדמות עליון** (`1/5..5/5`) — נקודות מתמלאות `accent` **מימין-לשמאל** (RTL).
- **"דלג"-עדין** (caption, `text-muted`, פינה-עליונה-שמאל) למתקדמים — שאלון-קצר במכוון (5 בלבד).
- **מסקוט רוני (curious)** מלווה למטה-בצד, מגיב לבחירות (micro-nod). קסדה-כתומה.

## 4. מעברים (Transitions)

**עיקרון-B:** StudiesGo = spring-overshoot-בכל-מקום (אנרגטי). כיוון-B מ**אפק את ה-overshoot** למעברים-מבניים → tween-ease מהוגן; spring שמור ל-micro-interactions-של-משוב (לחיצה, הצלחה) כדי לשמר את ה-"חיים" של StudiesGo בלי הקפצנות. זה מה שהופך פרימיום.

### 4.1 Easing + Timing קנוני (Framer-Motion)

```ts
// כיוון-B — מאופק-מהוגן
const easeOut = [0.22, 0.61, 0.36, 1]; // ease-out רגוע — מעברי-מסך/route
const easeBoth = [0.45, 0, 0.55, 1]; // ease-in-out — slide-onboarding
const springSoft = { type: 'spring', stiffness: 300, damping: 30 }; // micro (פחות-overshoot מ-StudiesGo 400/25)
const springPop = { type: 'spring', stiffness: 400, damping: 22 }; // pop-משוב מאופק (היה 500/15)

const durations = { fast: 0.12, base: 0.22, medium: 0.32, slow: 0.45 }; // מעט-איטי-יותר → תחושת-יוקרה
```

> **למה איטי-יותר-במעט:** מעבר של `.22-.32s` עם ease-out רגוע נקרא "מכובד/יקר". `.05-.15s` חד נקרא "מהיר/צעיר". כיוון-B בוחר את הראשון למבנה, שומר חד למשוב.

### 4.2 Page / Route transitions

| מעבר                        | spec                                                                                                                                     |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **route-החלפה**             | cross-fade + slide-עדין (`opacity 0→1`, `y 8→0`), `easeOut`, `base .22s`. **ללא** slide-מלא-מסך (מהוגן).                                 |
| **onboarding step→step**    | slide-אופקי-RTL (`X 100vw→0` נכנס, `0→-100vw` יוצא — היפוך-RTL), `easeBoth`, `medium .32s` + מד-התקדמות מתעדכן.                          |
| **dashboard → lesson**      | shared-element (ראה 4.3) + fade-תוכן.                                                                                                    |
| **theme-switch light↔dark** | cross-fade-רך של רקע+משטחים (`base .22s`, `easeOut`) — **לא** flash. (קריטי: ה-quiz light, skill-tree/stats dark — מעבר חייב להיות חלק). |

### 4.3 Shared-element transitions

| מקור→יעד                                   | אלמנט-משותף            | spec                                                                                     |
| ------------------------------------------ | ---------------------- | ---------------------------------------------------------------------------------------- |
| course-card (dashboard) → lesson-header    | כותרת-הקורס + צבע-עוגן | `layoutId="course-{id}"`, `springSoft`, morph-מיקום+גודל.                                |

- modal/dialog: `scale .96→1` + `opacity` + backdrop `blur 0→6px` (`springSoft`, היה scale .8 — מאופק).

- tooltip/popover: `opacity + y 4→0`, `fast .12s`, `easeOut`.
- accordion (settings-sections): `height auto` + `opacity`, `layout` spring.

## 7. מראה נקי / נעים / מזמין (Vibe + Empty/Loading/Error)

### 7.1 ה-vibe הכללי

- **רווח-לבן נדיב** (`space-8/10/12` בין-בלוקים) — החומר-הראשי. המסך נושם.
- **hairline-borders + צללים-רכים-מאוד** — עומק בלי-רעש.
- **צבע-נקודתי** — הכתום-accent מופיע במשורה → כל-הופעה "נשמעת". זה מה שהופך מזמין-בלי-להעמיס.
- **טיפוגרפיה-נושמת** — `line-height 1.6`, כותרות-מרווחות.
- **פינות-רכות** (`r-card 14` · pills) — ידידותי, לא-חד.
- **מסקוט-נוכח-מאופק** — חום-אנושי בלי-ילדותיות.
- **תנועה-מהוגנת** — ease-רגוע למבנה, spring-עדין למשוב.

> **התוצאה הרגשית:** "מישהו מקצועי הכין לי את זה בקפידה." מתאים ל**מוצר-לשיווק** שצריך לשדר ערך-מהשנייה-הראשונה.

## 8. RTL + נגישות (a11y / WCAG AA)

### 8.1 RTL (אזרח-ראשון)

- **logical-properties בלבד:** `ps-*`/`pe-*`/`ms-*`/`me-*`, `inset-inline`, `text-align: start` — לעולם לא `left`/`right` פיזי.
- **חיצים/chevrons מתהפכים:** "המשך →" מצביע **שמאלה** ב-RTL · back-arrow ימינה · chevron-accordion. (נצפה ב-`create-c`).
- **מד-התקדמות מתמלא מימין-לשמאל** (`●●○○○` — מלא מימין).
- **gradient-כפתורים אלכסוניים:** כיוון-זרימה תואם-RTL (`to-left` במקום `to-right`).
- **slide-onboarding היפוך-RTL:** נכנס מימין (`X 100vw→0` → בפועל מהצד-הימני), יוצא שמאלה.
- **shared-element + nav-pill:** כיווני-החלקה תואמי-RTL.
- **מספרים/תאריכים:** עברית — מספרים LTR בתוך-משפט-RTL (bidi-isolation `<bdi>`), תאריכים `dd/mm/yyyy`.
- **בדיקת-Playwright חובה תחת `dir="rtl"`** (חתימת-CLAUDE.md).

### 8.2 נגישות (יעד: axe-core 0-violations · Lighthouse a11y ≥95)

| דרישה                      | יישום-בכיוון-B                                                                                                                                                                                                                        |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ניגודיות AA**            | text-primary `#1A2233` על `#FBFCFE` ≈ 14.8:1 (AAA). `primary #1B4FD6` על לבן ≈ 6.4:1 (AA-טקסט-גדול ✓, AA-רגיל ✓). `accent #F5A623` — **לא** לטקסט-על-לבן (יחס-נמוך); משמש border/bg-בלבד או טקסט על-`accent-soft` כהה. כל-שילוב נבדק. |

| **focus-visible**          | `shadow-focus` (טבעת-כחולה 3px) על כל-אינטראקטיבי, נראה-בבירור על-light ו-dark.                                                                                                                                                       |
| **keyboard-nav מלא**       | Tab בין-שדות-auth · בין-controls-settings · בחירת-תשובות-quiz (חצים+Enter) · onboarding (Tab+Space). focus-trap ב-modals.                                                                                                             |

| **prefers-reduced-motion** | `respectReducedMotion()` → מאפס spring/shake/confetti/float → fade-בלבד או ללא. חובה (motion-specs).                                                                                                                                  |
| **mascot SVG**             | `role="img"` + `aria-label` תיאורי ("רוני, המדריך, מחייך"). pose-decorative → `aria-hidden`.                                                                                                                                          |

| **כפתור-נגישות גלובלי**    | toggle ב-settings (גודל-טקסט/ניגודיות-גבוהה/הפחתת-תנועה) — זמין-בכל-עמוד.                                                                                                                                                             |
| **מחק-חשבון**              | מחיקה-מלאה (GDPR right-to-be-forgotten) — אישור-כפול, danger-zone.                                                                                                                                                                    |

## 9. למה הכיוון מתאים למוצר-לשיווק (קהל ממוני-בטיחות עברי)

1. **כבוד לקהל-בוגר-ומקצועי.** ממוני-בטיחות הם אנשי-מקצוע (לרוב 35-60), רבים מרקע-הנדסי/ניהולי. מינימליזם-חם + טיפוגרפיה-נושמת + כתום-מאופק = "מערכת רצינית שמכבדת את הזמן והאינטליגנציה שלי" — לא אפליקציית-משחק-לילדים. זה **קריטי ל-conversion** בקהל-זה.

2. **אמינות = ערך-מכירה-מרכזי.** בתחום-הבטיחות, טעות = חיי-אדם וחבות-משפטית. הכיוון מטמיע **אמון בעיצוב**: `AiNotice` קבוע (שקיפות), badge-"מקור" עם citation מ-PDF (PDF-as-source-of-truth — CLAUDE.md), טון-רגוע-ומכובד. עיצוב-פרימיום-שקט משדר "אפשר לסמוך על המידע כאן" — נכס-מכירה ישיר.

3. **מראה-יקר = הצדקת-תמחור.** מוצר-לשיווק צריך להרגיש שווה-תשלום מהשנייה-הראשונה. רווח-לבן, צללים-רכים, מעברים-מהוגנים, ופלטה-מאופקת = שפת-העיצוב של מוצרי-SaaS-פרימיום (Notion/Linear/Stripe). זה **מצדיק מחיר** מול-מתחרים-זולים-נראים.

4. **שמירת-DNA-StudiesGo + בידול.** הכיוון משמר את חתימת-StudiesGo (כחול↔כתום · pill · flat-cards · glass-nav · mascot-מרחף · spring) — בסיס-מוכח. ה**בידול-לשיווק** (mascot "רוני" אנושי + קסדת-בטיחות-תמטית + מסלול-אישי-מותאם + עברית-RTL-נטיבית-מושלמת) הופך אותו ל**מוצר-עברי-ייחודי**, לא-קלון. עברית-RTL-מושלמת היא יתרון-תחרותי-עצום בשוק-מקומי שרגיל ל-UI-מתורגם-גרוע.

5. **רגיעה מפחיתה-חרדת-מבחן.** הקהל לומד לקראת ועדת-הסמכה לחוצה. אסתטיקה-רגועה (לא-עמוסה, לא-מבהבת, gamification-מאופק) **מפחיתה-עומס-קוגניטיבי** ותומכת-בריכוז-בלמידה — ערך-מוצר-אמיתי, לא רק-נראות.

6. **התאמה-מושלמת-לתוכן-הבטיחות.** טון-מהוגן + קסדת-בטיחות-כתומה כמוטיב-חזותי + מסלול-אישי-עד-הוועדה = הכל-מתלכד לזהות-"הליווי-המקצועי-שלך-לוועדה". העיצוב **מספר את סיפור-המוצר** — חזק לשיווק.
