## חלק א' — Timeline פר-פריים מלא

עבור על-כל-הסרטון מ-00:00.000 עד-הסוף, ותן רישום-מדויק כזה:

00:00.000 — [תיאור-מה-נראה: איזה-מסך, איזה-אלמנטים, מה-הפוזה של המסקוט]
00:00.100 — [מה-השתנה ב-100ms האחרונים? אם-כלום, כתוב "ללא-שינוי"]
00:00.200 — [המשך...]
...

חוקי-תיאור:
- כל-פעם שמשהו זז: ציין מאיפה לאן (px משוערים), באיזה direction
- כל-פעם שמשהו משנה צבע: ציין hex-from → hex-to משוערים
- כל-פעם שמשהו משנה גודל (scale): ציין מ-1.0 ל-1.X
- כל-פעם שמשהו מופיע/נעלם: ציין opacity 0→1 או 1→0 + משך
- ציין את ה-easing המשוער: linear / ease-out / ease-in-out / spring-bouncy / spring-stiff

## חלק ב' — Deep-dives לאירועים-קריטיים

לאחר ה-timeline-המלא, חזור על-כל אירוע-קריטי בנפרד עם פירוט-של-50ms:

1. **תשובה-נכונה** (אם יש בסרטון):
   - 50ms-לפני-אישור: מצב-מסך
   - רגע-אישור (T=0): מה השתנה ראשון?
   - T+50ms: ...
   - T+100ms: ...
   - ... עד T+1500ms או עד המסך-הבא
   - ציין: סוג-easing, צבעים, scale, opacity, transforms

2. **תשובה-שגויה** (אם יש):
   - אותו ניתוח של 50ms-frames

3. **מעבר-בין-שאלות**:
   - אותו ניתוח

4. **כל אנימציה-מיוחדת-נוספת** (XP-counter, streak-fire, robot-pose-change, וכו'):
