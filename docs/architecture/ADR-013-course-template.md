## Context (למה צריך החלטה)

StudiBuilder הוא Course-as-Product Factory (ADR-006): מוטי מייצר קורסים, וכל קורס "נולד"
כמוצר. כדי שזה יהיה scaleable, חייבת להיות **תבנית-קורס אחת** שכל קורס יורש — אחרת כל קורס
ייבנה מאפס, וה-Learning-area (Phase 5) יצטרך לטפל במבנים שונים פר-קורס.

הקורס הראשון, "ממונה בטיחות", כבר התגבש סביב דפוס חוזר:

- spine של פרקים (11 פרקי-אתגר), כל פרק מכסה אשכול-נושאים מתוך מסנן ה-57-scope.
- כל פרק נלמד בשלושה מצבי-למידה משלימים: **שו"ת** (שאלות-תשובות), **תרחישים** (case-based),
  ו-**מורה-AI** (tutor שיחתי). שלושת המצבים יושבים על אותו תוכן-פרק.
- **ניווט-כפול**: אפשר לנווט לפי ה-spine (פרק-אחרי-פרק, מסלול-מודרך) או לפי המסנן (קפיצה
  ישירה ל-scope ספציפי, מסלול-חופשי).
- בקצה הקורס יש **capstone** אופציונלי — מיני-קורס נוסף שמלווה הפקת תוצר-אמיתי (בקורס
  ממונה-בטיחות: עבודת-גמר אמיתית על מפעל/אתר, דרישת משרד-העבודה).

ללא קודיפיקציה, הדפוס הזה נשאר ידע-מובלע בקורס-אחד. צריך להפוך אותו לחוזה.

---

## Decision (מה הוחלט)

**תבנית-הקורס הקנונית: "פרק = מיני-קורס × 3 מצבים + ניווט-כפול".**

כל קורס בפלטפורמה מורכב מ-spine של **פרקים**. כל פרק הוא **מיני-קורס** עצמאי המוגש בשלושה
מצבי-למידה על אותו גוף-תוכן, וניתן-להגעה בשני צירי-ניווט. בנוסף, קורס יכול לכלול
**מיני-קורס-capstone** אופציונלי (פרויקט-גמר) מעבר ל-3 המצבים.

**Pattern שנאמץ**: מבנה היררכי `קורס → פרק (מיני-קורס) → {שו"ת | תרחישים | מורה-AI} (+ capstone)`.
**מה זה מחליף**: בנייה אד-הוק פר-קורס.

### מודל שלושת-המצבים (לכל פרק)

| מצב-למידה | מהות                                         | סוג-אינטראקציה     |
| --------- | -------------------------------------------- | ------------------ |
| שו"ת      | שאלות-תשובות ממוקדות-scope (כולל בנקי-שאלות) | quiz / תרגול-מובנה |
| תרחישים   | מקרי-בוחן יישומיים (case-based)              | החלטה-בהקשר        |
| מורה-AI   | tutor שיחתי על תוכן-הפרק (persona, ADR-009)  | דיאלוג חופשי       |

### ניווט-כפול

- **ציר-spine** — מסלול-מודרך, פרק-אחרי-פרק (לומד-חדש).
- **ציר-מסנן** — קפיצה ישירה ל-scope/נושא (לומד-ממוקד / חזרה לפני בחינה).

### מיני-קורס-capstone (אופציונלי)

מעבר ל-3 המצבים, קורס יכול לכלול capstone שמלווה הפקת **תוצר-אמיתי** צעד-אחרי-צעד. בקורס
ממונה-בטיחות זהו ליווי עבודת-גמר אמיתית (בחירת-אתר → סקר-מפגעים → הערכת-סיכונים →
תכנית-לניהול-הבטיחות → המלצות → תיעוד). ה-capstone הוא מצב-יצירה, לא צריכה.

**מקרא**: ✅ חלק-מהתבנית-החובה · 🟠 אופציונלי-פר-קורס · 🔴 לא-בתבנית.

| רכיב-תבנית           | סטטוס | הערה                                     |
| -------------------- | ----- | ---------------------------------------- |
| spine של פרקים       | ✅    | כל קורס                                  |
| 3 מצבי-למידה לכל פרק | ✅    | שו"ת · תרחישים · מורה-AI                 |
| ניווט-כפול           | ✅    | spine + מסנן                             |
| מיני-קורס-capstone   | 🟠    | פרויקט-גמר; חובה רק היכן שהרגולציה דורשת |

### אכיפה (oversight)

חוזה תיקיית-`courses/<slug>/` ומודל-3-המצבים אינם רק תיעוד — הם **נאכפים** ע"י **ענף-הבקרה**
(`oversight` — בהקמה, סשן-B; ראה תוכנית-ממשל-v2). שלוש בדיקות-אובייקטיביות:

- **גבול-שכבות:** ידע-קורס (chapter/legislation/PROGRAM/scope ספציפי-לקורס) שמחלחל ל-`src/` =
  **הפרת-גבול** → דגל. הפלטפורמה נשארת קורס-אגנוסטית; הקורס יושב ב-`courses/<slug>/` בלבד
  (ראה `../PROJECT-STRUCTURE.md` §גבול-השכבות).
- **תאימות-תבנית:** קורס שאינו עומד בחוזה (חסר קובץ-חובה / פרק ללא 3-מצבים / ניווט-יחיד) = דגל.
- **כיסוי מול PROGRAM:** מבקר-התכנית מצליב את ה-spine/scope מול תכנית-הרגולטור הרשמית
  (קורס #1: `MOLSA-PROGRAM.md` מול 905018); חוסר-כיסוי או תוכן out-of-curriculum = דגל.

> צו-עצירה (אם נדרש) נתון לקוורום-בקרה 2/3 + ראיה ב-ledger; רק-מוטי מבטל. (פירוט: פרוטוקול-הבקרה, סשן-B.)

## Alternatives Considered (לא בחרנו)

### Option A: שיעורים-שטוחים (flat lessons)

רשימת-שיעורים לינארית בלבד, ללא מיני-קורס פר-פרק וללא מצבי-למידה.

- ✅ פשוט-לבנייה ולמודל-נתונים
- ❌ אין דרכי-למידה שונות (שו"ת מול תרחישים מול tutor) — חוויית-למידה דלה
- ❌ אין ניווט-ממוקד-scope — לומד-חזרה חייב לדפדף לינארית

### Option B: מצב-יחיד (single-mode)

פרקים עם דרך-למידה אחת בלבד (למשל רק quiz, או רק tutor).

- ✅ קל יותר לבנות ולתחזק מבחינת UI
- ❌ מפספס שלמדנים שונים זקוקים למצבים שונים; פוגע ב-retention
- ❌ לא ממנף את נכסי-התוכן הקיימים (בנקי-שאלות + תרחישים + persona) שכבר קיימים יחד

### Option C: בנוי-לכל-קורס (per-course bespoke)

כל קורס מתוכנן ונבנה במבנה משלו.

- ✅ התאמה-מקסימלית פר-נושא
- ❌ מנוגד ל-Factory (ADR-006) — כל קורס = פרויקט-פיתוח חדש, לא scaleable
- ❌ ה-Learning-area (Phase 5) יצטרך לתמוך ב-N מבנים שונים

**למה לא A/B/C**: A ו-B פוגעים בחוויית-הלמידה ואינם ממנפים את מגוון-הנכסים; C מנוגד למודל
ה-Factory ולא-scaleable. תבנית "פרק=מיני-קורס×3-מצבים+ניווט-כפול" מאזנת חוויה-עשירה עם
ירושה-אחידה שמאפשרת ייצור-קורס-חדש כמילוי-תבנית.

## Consequences

### Positive

- כל קורס **יורש את חוזה `courses/<slug>/`** ואת מודל-3-המצבים — ייצור-קורס-חדש = מילוי-תבנית
- ה-Learning-area (Phase 5) נבנה פעם-אחת מול תבנית אחת, לא פר-קורס
- חוויית-למידה עשירה: שלושה מסלולי-למידה + שני צירי-ניווט מתוך אותו תוכן
- capstone אופציונלי נותן מענה לדרישות-רגולטוריות (כמו עבודת-גמר אמיתית) בלי לשבור את התבנית

### Negative / Trade-offs

- מודל-נתונים עשיר יותר — צריך לתמוך ב-3 מצבים + 2 צירי-ניווט + capstone (ADR-010/011)
- כל קורס חייב לספק תוכן ל-3 המצבים (לא רק שיעור-אחד) — נטל-תוכן גבוה יותר ביצירה
- migration-path: קורס שלא מתאים לתבנית (למשל קורס-מיקרו) ידרוש ADR-משלים או "מצב מצומצם"
  של התבנית

### Neutral

- שינויים נדרשים: סכמת-ה-DB (ADR-010) וצינור-הייבוא (ADR-011) צריכים לשקף פרק→מצבים→capstone
- `COURSE-DESIGN.md` ו-`PROJECT-STRUCTURE.md` מתעדים את היישום הקונקרטי של התבנית

## Implementation Plan

1. לקבע את התבנית כחוזה ב-`docs/PROJECT-STRUCTURE.md` (חוזה תיקיית `courses/<slug>/`).
2. למפות את קורס ממונה-בטיחות (instance #1) לתבנית ב-`COURSE-DESIGN.md` — 11 פרקים × 3 מצבים.
3. לוודא שסכמת-ה-DB (ADR-010) וצינור-הייבוא (ADR-011) תומכים בפרק→מצבים→ניווט-כפול→capstone.
4. לבנות את ה-Learning-area (Phase 5) מול התבנית — `<LessonPlayer>` עם 3 מצבים.
5. להוסיף checklist "קורס-עתידי" ל-`PROJECT-STRUCTURE.md` לבדיקת-תאימות-תבנית.

**שעות עבודה מוערכות**: התבנית עצמה — 0 (קודיפיקציה של דפוס-קיים); יישום ב-Learning-area =
חלק מ-Phase 5.
**Phase שבו זה ייעשה**: 4-5 (תוכן + Learning-area), קיבוע ב-Phase 10 (Factory).
