## סכמת שורת-מטא מורחבת

כל תת-משימה **פתוחה** נושאת **שורת-מטא** מתחתיה — אומדן-מאמץ + סיכון + בעלות + שער-אימות. כך
**ענף-הבקרה משתמש בזה כ-checklist-אובייקטיבי** (oversight) — לבדוק התקדמות מול **checklist אובייקטיבי**
ולא מול תחושה.

```
- [ ] **<ID> · <כותרת>** — <תיאור> · קריטריון-קבלה: … · ref: …
  - 📊 **מטא:** ⏱<h>h · 🤖<n>(<slugs>) · 💲~$<range> · 🟢|🟡|🔴 · ראש-צוות:<lead> · <🚩דורש|—> · אימות:<Workflow|solo>
```

**מקרא השדות:**

| שדה              | משמעות                                                                                                                    |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------- |
| ⏱`<h>h`          | אומדן-מאמץ בשעות (גס; לכיול בעת-הביצוע)                                                                                   |
| 🤖`<n>(<slugs>)` | מספר-הסוכנים שצפויים לגעת + ה-slugs שלהם (מתוך הרוסטר — [`../../{{TEAMS_DIR}}/README.md`](../../{{TEAMS_DIR}}/README.md)) |
| 💲`~$<range>`    | טווח-עלות-AI צפוי (`$0` = קוד/תיעוד; חריגה מ-hard-cap = דגל-בקרה)                                                         |
| 🟢/🟡/🔴         | סיכון: 🟢 שגרתי · 🟡 עלות/אינטגרציה/ציות · 🔴 כספי/אבטחה/רגולציה                                                          |
| ראש-צוות         | ה-lead האחראי (מתוך {{TIERS}} — ראשי-הצוות בפרויקט)                                                                       |
| 🚩דורש           | חוסם על אישור-{{COUNCIL_MEMBER}} מפורש (`—` = לא-חוסם)                                                                    |
| אימות            | `Workflow` (code-review/security-review) למשימות-קוד מהותיות · `solo` לתיעוד/החלטה                                        |

> **שימוש-הבקרה:** השדות הם ה-checklist האובייקטיבי של ענף-הבקרה:
> ✅-ללא-`activity-log` → דגל · 🚩-ללא-אישור-{{COUNCIL_MEMBER}} → צו-עצירה · חריגת-💲/🔴-ללא-`Workflow` → דגל.
> תתי-משימות **שהושלמו** (✅) נושאות מטא-מינימלי (✅ + ראש-צוות) — לא רטרואקטיבי-מלא.
