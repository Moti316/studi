## 3 מצבי-הלמידה

מודל-הליבה: כל פרק (מיני-קורס) נלמד דרך **3 מצבים משלימים**. כל מצב ממופה לאחת מ-3
**פעולות-הוועדה** שהבוחן מצפה להן בבחינה העיונית והבעל-פה — **זיהוי · הסבר · יישום**. שלושת
המצבים יחד מכסים את מעבר-הלומד מ"מכיר את החומר" ל"מסוגל לפעול כממונה".

| מצב         | פעולת-הוועדה | מה הלומד עושה                                | תוצר-למידה                        |
| ----------- | ------------ | -------------------------------------------- | --------------------------------- |
| שו"ת        | זיהוי        | עונה על שאלות-ידע ממוקדות לפי scope ופרק     | שליפה מהירה + סגירת-פערים         |
| תרחישים     | יישום        | מקבל החלטה במצב-שדה מדומה ומנמק לפי החקיקה   | שיקול-דעת יישומי תחת אילוץ        |
| מורה-AI/RAG | הסבר         | שואל בחופשיות; מקבל הסבר מעוגן-מקור עם ציטוט | הבנה לעומק + יכולת-הנמקה מול ועדה |

**מקרא:** ✅ מוטמע · 🟠 חלקי/בבנייה · 🔴 חסר.

### שו"ת (פעולת-הוועדה: זיהוי)

מצב תרגול-ידע מובנה. הלומד עונה על שאלות נבחרות מבנק-השאלות, מסוננות לפי **פרק** ולפי
**scope-ID**, עם משוב מיידי ומעקב-שליטה פר-נושא. המצב מאמן את **הזיהוי** — היכולת לקשר
מצב/מושג לסעיף-החקיקה או לתקן הרלוונטי, שהיא דרישת-הליבה בבחינה העיונית-הכתובה (40%, עובר 70).

- **מקור-תוכן:** בנק-השאלות (~600+ שאלות, 18 קבצים) — ראה `LEARNING-MATERIALS.md`. כל שאלה
  מתויגת ל-scope ול-פרק לצורך הסינון-הכפול.
- **כיסוי:** ✅ לרוב הפרקים; 🟠 פערי-בנק בנושאים image-only/סימולציה (OCR לתיקון — ראה
  `UNREAD-MEDIA.md`).
- **כלל-מגן:** כל שאלה מסומנת `in_scope` בלבד מוזרמת ל-quiz (ראה `../../docs/content-scope.md`).

### תרחישים / סימולציות (פעולת-הוועדה: יישום)

מצב יישומי. הלומד מוצב במצב-שדה מדומה (אתר-בנייה, מפעל, מעבדה, עבודה-בגובה) ונדרש **להחליט
ולנמק** — לזהות מפגע, לבחור בקרה לפי מדרג-הבקרות, ולהצביע על הסעיף שמחייב את הפעולה. זהו
המצב שמדמה את **הבחינה בעל-פה מול ועדת-הבוחנים** (60%, עובר 60).

- **תרחיש-דגל:** "כתוב תכנית-לניהול-הבטיחות" — הלומד בונה תכנית בהתאם ל-scope **1.2** (תקנות
  ארגון-הפיקוח (תכנית לניהול הבטיחות), תשע"ג-2013), נשען על תהליך סקר-סיכונים / מטריצה / מדרג-
  בקרות / JSA. זוהי כשירות-ליבה שהוועדה בודקת, וגשר ישיר אל מצב-היצירה (capstone) שלמטה.
- **מקור-תוכן:** שאלות-סימולציה + שאלות-בחינת-וועדה (פתוחות) + מצגות ניהול-הסיכונים.
- **כיסוי:** ✅ ניהול-סיכונים ובנייה; 🟠 תרחישי-עזרה-ראשונה (2.5) וגז/גפ"מ (4.5) — לחיזוק.

### מורה-AI / RAG (פעולת-הוועדה: הסבר)

מצב שיחה-חופשית. הלומד שואל בשפה-טבעית, והמורה-AI מחזיר **הסבר מעוגן-מקור** (Retrieval-
Augmented Generation) עם ציטוט הסעיף/התקן ומצב-תשובה מפורש. המצב מאמן את ה**הסבר** — היכולת
לנסח נימוק מלא ומדויק כפי שהוועדה מצפה, ולא רק לזהות תשובה נכונה.

- **עיגון-מקור:** תשובות נשענות על קורפוס-החקיקה (נחלת-כלל, שימוש-ישיר — ראה
  `LEGISLATION-SOURCES.md`) ועל תקצירי-ISO מקוריים (5.x — בתשלום, תקצירים-בלבד). לא משוכפל
  טקסט-תקן.
- **מצבי-תשובה:** `[מאומת]` / `[מוסקנא]` / `[לא ידוע]` — לפי מתודולוגיית-מגן (ראה
  `## מתודולוגיית-מגן`).
- **כיסוי:** 🟠 בבנייה — תלוי בפייפליין-הייבוא (ADR-011) ובהטמעת-RAG.

## מצב-יצירה (capstone) — פרויקט-גמר

מעבר ל-3 מצבי-הלמידה קיים **מצב רביעי — מצב-יצירה**: המערכת **מלווה את הלומד בהפקת עבודת-גמר
אמיתית** על מפעל/אתר-בנייה אמיתי (דרישת משרד-העבודה — לא תרגיל פיקטיבי). בעוד 3 המצבים
הראשונים מאמנים זיהוי/הסבר/יישום על חומר נתון, מצב-היצירה דורש מהלומד **לייצר תוצר-מקצועי
חדש** — ובכך מבטא את הרמה הגבוהה ביותר של כשירות-הוועדה.

התוצר המרכזי הוא **תכנית-לניהול-הבטיחות אמיתית** (scope **1.2**, תשע"ג-2013) עבור אתר שהלומד
בוחר. רצף-ההפקה: בחירת-אתר-אמיתי → סקר-מפגעים → הערכת-סיכונים (JSA/מטריצה) → תכנית-לניהול-
הבטיחות → המלצות → תיעוד. המערכת מלווה שלב-אחר-שלב, נשענת על מצב מורה-AI להסבר ועל מצב-
תרחישים לתרגול-ביניים.

- **מבנה מלא + רצף-ההנחיה:** `./FINAL-PROJECT.md`.
- **גשר ממצב-תרחישים:** תרחיש-הדגל "כתוב תכנית-לניהול-הבטיחות" הוא חזרה-מבוקרת לקראת מצב-
  היצירה.
- **תמיכת-תבנית:** התבנית תומכת ב-מיני-קורס-capstone כסוג-פרק נוסף (ראה
  `../../docs/architecture/ADR-013-course-template.md`).

**מקרא:** ✅ מוגדר במבנה · 🟠 מצגת-הנחיות-מדויקת + פרויקט-דוגמה — TODO (ראה `./FINAL-PROJECT.md`).

## תבנית-הפלטפורמה

המבנה אינו ייחודי לקורס-זה — הוא **תבנית-הפלטפורמה הניתנת-לשימוש-חוזר** לכל קורס עתידי
(תואם Course-as-Product Factory — `../../docs/architecture/ADR-006-course-as-product-factory.md`).
העיקרון: **פרק = מיני-קורס × מצבים**, עם **ניווט-כפול** ו**ייצור היברידי**.

- **פרק = מיני-קורס:** כל אחד מ-11 הפרקים הוא יחידת-למידה עצמאית, שעוברת דרך 3 המצבים + (בפרק
  המתאים) מצב-היצירה.
- **ניווט-כפול:**
  - **לפי-פרק** — הלומד בוחר פרק ועובר את כל מצביו (מסלול-עומק נושאי).
  - **לפי-מצב** — הלומד בוחר מצב (למשל "שו"ת") וחותך אותו אופקית על-פני כל הפרקים
    (מסלול-תרגול רוחבי).
- **ייצור היברידי:** התוכן מיוצר בשילוב מקורות-אדם משוכתבים (חומרי-מרצה → reference + שכתוב-
  מהותי), טקסט-חקיקה (נחלת-כלל), ותקצירי-ISO מקוריים — מוזרם דרך פייפליין-הייבוא (ADR-011)
  אל סכמת-הנתונים (ADR-010). מבנה התבנית מתועד ב-`../../docs/architecture/ADR-013-course-template.md`.
- **שימוש-חוזר:** חוזה תיקיית-הקורס (`courses/<slug>/`) + מיפוי-התיקייה-ל-DB מתועדים ב-
  `../../docs/PROJECT-STRUCTURE.md`, כך שקורס-עתידי נבנה באותו מבנה ללא שכפול-עיצוב.

| ציר-תבנית     | מימוש בקורס-זה                                             |
| ------------- | ---------------------------------------------------------- |
| פרק=מיני-קורס | 11 פרקי `curriculum-atgar.md`                              |
| מצבים         | שו"ת · תרחישים · מורה-AI + capstone                        |
| ניווט-כפול    | לפי-פרק / לפי-מצב                                          |
| ייצור היברידי | מקורות-אדם משוכתבים + חקיקה + תקצירי-ISO → ADR-011→ADR-010 |
| שימוש-חוזר    | חוזה `courses/<slug>/` — `../../docs/PROJECT-STRUCTURE.md` |
