# ISO/IEC 31010 (ת"י 31010) — שיטות הערכת-סיכונים (טיוטה — לסקירת מוטי)

> טיוטה לסקירת מוטי · תקציר-מקורי · אינו תחליף לתקן הרשמי בתשלום · מעודכן 2026-06-01

> **מטרה:** להציג את ISO/IEC 31010 כ**קטלוג של שיטות הערכת-סיכונים** (Risk Assessment
> Techniques) שכבר נלמדות בקורס ממונה-בטיחות — ולמפות לכל שיטה מתי משתמשים בה, יתרונותיה
> וחסרונותיה. התקן הוא הנספח-הטכני של ISO 31000: 31000 מגדיר את **תהליך** ניהול-הסיכונים,
> ו-31010 מספק את **הכלים** לשלב ההערכה (זיהוי → ניתוח → הערכה).
> **מקור:** טיוטה זו נכתבה כ**תקציר-מקורי** הנשען על השיטות הנלמדות בקורס (מצגות
> ניהול-הסיכונים, סיכומי-שיעור, בנקי-השאלות). **לא שוכפל ולא צוטט טקסט מתוך התקן הרשמי**
> (ISO/IEC 31010 — בתשלום, אינו נחלת-הכלל). שייכות: scope **5.3** (`ת"י31010`).
> **עוגנים:** [`ISO-31000-DRAFT.md`](ISO-31000-DRAFT.md) · [`LEGISLATION-SOURCES.md`](LEGISLATION-SOURCES.md) ·
> [`LEGISLATION-COVERAGE.md`](LEGISLATION-COVERAGE.md) · כלל-זכויות: [`ATTRIBUTION.md`](ATTRIBUTION.md) ·
> [`../../docs/compliance/COMPLIANCE.md`](../../docs/compliance/COMPLIANCE.md).

## על מה מדובר

ISO/IEC 31010 אינו מטיל דרישות מערכת-ניהול ואינו תקן בר-הסמכה. הוא **מסמך תומך**
שמתאר מאגר רחב של שיטות מובְנות להערכת-סיכונים, ומסביר באילו נסיבות כל שיטה יעילה.
מבחינת ממונה-הבטיחות התקן הוא בעצם "ארגז-הכלים" המסודר של כל מה שכבר נלמד בקורס תחת
הכותרת ניהול-סיכונים: כיצד מזהים מפגעים באופן שיטתי, כיצד מנתחים את הסבירות והחומרה,
וכיצד מחליטים מה מספיק חמור כדי לדרוש בקרה.

הרעיון המרכזי: **אין שיטה אחת נכונה**. בחירת השיטה תלויה בשלב התהליך (זיהוי / ניתוח /
הערכה), במורכבות המערכת, באיכות הנתונים הזמינים ובמשאבים. ממונה-בטיחות מיומן יודע לבחור
את הכלי המתאים — ולעיתים לשלב כמה כלים יחד באותו סקר-סיכונים.

## קטלוג השיטות הנלמדות בקורס

לכל שיטה: תיאור-קצר · מתי מתאימה · יתרון מרכזי · חיסרון מרכזי. כל התיאורים מנוסחים
מחדש במילותינו על-בסיס החומר הנלמד בקורס.

### JSA — ניתוח בטיחות-משימה (Job Safety Analysis)

| היבט         | פירוט                                                                                 |
| ------------ | ------------------------------------------------------------------------------------- |
| תיאור-קצר    | פירוק עבודה/משימה לשלביה הרצופים, זיהוי המפגע בכל שלב, וקביעת בקרה לכל מפגע.          |
| מתי מתאימה   | משימות-עבודה מוגדרות וחוזרות (תחזוקה, עבודה-בגובה, כניסה-למוקפים) · בסיס לנוהל-עבודה. |
| יתרון מרכזי  | פשוטה, מעשית, מחברת ישירות בין מפגע לבקרה ולהדרכת-העובד.                              |
| חיסרון מרכזי | תלוית-משימה — אינה מתאימה לניתוח מערכת מורכבת או לכשלים מרובי-גורמים בו-זמנית.        |

### What-if / Checklist — "מה אם" ורשימת-תיוג

| היבט         | פירוט                                                                              |
| ------------ | ---------------------------------------------------------------------------------- |
| תיאור-קצר    | סבב שאלות מובְנה ("מה אם...?") בליווי רשימת-תיוג שיטתית לזיהוי תרחישי-כשל אפשריים. |
| מתי מתאימה   | שלב-זיהוי מוקדם · תהליכים פשוטים-בינוניים · סקירה רוחבית מהירה לפני העמקה.         |
| יתרון מרכזי  | זול, מהיר, מעודד חשיבה יצירתית בצוות; הרשימה מבטיחה שלא נשכח נושאים.               |
| חיסרון מרכזי | איכותו תלויה בניסיון המשתתפים ובשלמות הרשימה; עלול לפספס תרחישים שאינם ברשימה.     |

### PHA — ניתוח-מפגעים מקדים (Preliminary Hazard Analysis)

| היבט         | פירוט                                                                          |
| ------------ | ------------------------------------------------------------------------------ |
| תיאור-קצר    | סקירה ראשונית, בשלב מוקדם, של המפגעים העיקריים והתרחישים החמורים שדורשים בקרה. |
| מתי מתאימה   | תכנון מתקן/תהליך חדש · מעט נתונים זמינים · בסיס להחלטה אילו אזורים לנתח לעומק. |
| יתרון מרכזי  | מאתר סיכונים קריטיים מוקדם, כשתיקון עדיין זול; מכוון את ההערכה המעמיקה.        |
| חיסרון מרכזי | גסה ולא-ממצה מטבעה; אינה תחליף לניתוח מפורט בשלבים מתקדמים.                    |

### FMEA / FMECA — ניתוח אופני-כשל והשפעותיהם

| היבט         | פירוט                                                                                                      |
| ------------ | ---------------------------------------------------------------------------------------------------------- |
| תיאור-קצר    | בחינה רכיב-אחר-רכיב: כיצד כל רכיב עלול לכשול, מה ההשפעה, ודירוג לפי חומרה/סבירות/גילוי (FMECA = +קריטיות). |
| מתי מתאימה   | מערכות וציוד (מכונות, מתקנים) · עדיפות לתחזוקה ולתכנון-בקרות לפי ציון-סיכון (RPN).                         |
| יתרון מרכזי  | מקיף ושיטתי ברמת-הרכיב; מתעדף טיפול לפי חומרה.                                                             |
| חיסרון מרכזי | עתיר-זמן; מתמקד בכשל-בודד ומתקשה לתפוס שילובי-כשלים או גורמי-אנוש.                                         |

### HAZOP — מחקר תפעוליות-וסיכונים (Hazard and Operability Study)

| היבט         | פירוט                                                                                            |
| ------------ | ------------------------------------------------------------------------------------------------ |
| תיאור-קצר    | צוות רב-תחומי עובר על תהליך נקודה-אחר-נקודה ומיישם מילות-מפתח (יותר/פחות/הפוך/ללא) לזיהוי סטיות. |
| מתי מתאימה   | תהליכים רציפים ומורכבים (כימי, צנרת, לחץ, חומרים-מסוכנים) · מתועד גם ב-IEC 61882 (scope 5.5).    |
| יתרון מרכזי  | יסודי ומובְנה מאוד; חושף סטיות שלא חושבים עליהן אינטואיטיבית; כוח הצוות הרב-תחומי.               |
| חיסרון מרכזי | יקר וארוך; דורש מנחה מנוסה ותרשימי-תהליך מדויקים.                                                |

### FTA — ניתוח עץ-כשל (Fault Tree Analysis)

| היבט         | פירוט                                                                                              |
| ------------ | -------------------------------------------------------------------------------------------------- |
| תיאור-קצר    | שיטה דדוקטיבית "מלמעלה-למטה": מתחילים מאירוע-תקלה לא-רצוי ומפרקים אחורה לגורמים בעזרת שערי AND/OR. |
| מתי מתאימה   | חקר שורש-תקלה · מערכות בהן צירוף גורמים מוביל לכשל · ניתוח כמותי של הסתברות-כשל.                   |
| יתרון מרכזי  | תופס שילובי-כשלים וכשלים-משותפים; תומך בכימות הסתברותי.                                            |
| חיסרון מרכזי | ממוקד באירוע-עליון אחד בכל פעם; דורש מומחיות ונתוני-אמינות לכימות.                                 |

### ETA — ניתוח עץ-אירוע (Event Tree Analysis)

| היבט         | פירוט                                                                                                  |
| ------------ | ------------------------------------------------------------------------------------------------------ |
| תיאור-קצר    | שיטה אינדוקטיבית "מלמטה-למעלה": מאירוע-פותח מתחילים, ובוחנים את שרשרת ההצלחה/כשל של הבקרות עד התוצאות. |
| מתי מתאימה   | בחינת יעילות מחסומי-מיגון · ניתוח השלכות אפשריות של אירוע · משלים את FTA.                              |
| יתרון מרכזי  | ממפה את כל מסלולי-התוצאה האפשריים ואת תרומת כל מחסום-מיגון.                                            |
| חיסרון מרכזי | מתאים לאירוע-פותח בודד; עץ מסועף הופך מהר למורכב ולקשה-לניהול.                                         |

### Bow-Tie — ניתוח עניבת-פרפר

| היבט         | פירוט                                                                                                           |
| ------------ | --------------------------------------------------------------------------------------------------------------- |
| תיאור-קצר    | תרשים המשלב את צד-הגורמים (כעין FTA) עם צד-התוצאות (כעין ETA) סביב אירוע-מרכזי, ומציג את מחסומי-המניעה והשיקום. |
| מתי מתאימה   | תקשורת-סיכון לסיכונים מהותיים · הצגת מערך-בקרות שלם להנהלה ולעובדים · scope 6.4.                                |
| יתרון מרכזי  | ויזואלי ובהיר; מראה בבת-אחת גורמים, תוצאות ומחסומים — מצוין להדרכה ולקבלת-החלטות.                               |
| חיסרון מרכזי | איכותי בעיקרו (פחות כמותי); מפשט מציאות מורכבת לטובת בהירות.                                                    |

### ניתוח 5M — חמש קבוצות-הגורמים

| היבט         | פירוט                                                                                                                     |
| ------------ | ------------------------------------------------------------------------------------------------------------------------- |
| תיאור-קצר    | מסגרת-זיהוי הממיינת מקורות-סיכון לחמש קבוצות: אדם (Man), מכונה (Machine), חומר (Material), שיטה (Method), סביבה (Milieu). |
| מתי מתאימה   | זיהוי שיטתי בסקר-מפגעים · חקירת-תאונה · ודוא שלא נשכחה קטגוריית-גורם.                                                     |
| יתרון מרכזי  | פשוטה, זכירה, מבטיחה כיסוי רוחבי של כל סוגי-הגורמים.                                                                      |
| חיסרון מרכזי | מסגרת-מיון בלבד — אינה מכמתת סיכון ואינה תחליף לניתוח מעמיק.                                                              |

## מתי-משתמשים-במה — מטריצת-בחירה

מיפוי השיטה לשלב בתהליך ניהול-הסיכונים (לפי ISO 31000) ולסוג-הבעיה:

| צורך / שלב                        | שיטות מומלצות                | הערה                                       |
| --------------------------------- | ---------------------------- | ------------------------------------------ |
| זיהוי-מפגעים ראשוני, רוחבי        | What-if/Checklist · 5M · PHA | זול ומהיר; פותח את הסקר.                   |
| ניתוח משימת-עבודה חוזרת           | JSA                          | מחבר מפגע→בקרה→נוהל→הדרכה.                 |
| ניתוח מערכת/ציוד ברמת-רכיב        | FMEA / FMECA                 | מתעדף לפי ציון-סיכון (RPN).                |
| תהליך רציף מורכב / חומרים-מסוכנים | HAZOP                        | צוות רב-תחומי + מילות-מפתח (גם IEC 61882). |
| חקר גורמי-שורש לתקלה              | FTA                          | דדוקטיבי; תופס שילובי-כשלים.               |
| בחינת יעילות מחסומי-מיגון ותוצאות | ETA                          | אינדוקטיבי; משלים FTA.                     |
| תקשורת-סיכון מהותי להנהלה/עובדים  | Bow-Tie                      | ויזואלי; גורמים+תוצאות+מחסומים יחד.        |
| כימות הסתברותי                    | FTA · ETA · FMECA            | דורש נתוני-אמינות ומומחיות.                |

**כלל-אצבע:** מתחילים בשיטה רחבה וזולה לזיהוי (Checklist/5M/PHA), עוברים לשיטה ממוקדת
לניתוח (JSA/FMEA/HAZOP), ומשתמשים ב-FTA/ETA/Bow-Tie כאשר נדרשת העמקה בגורמים, בתוצאות
או בתקשורת-הסיכון.

## הקשר ל-ISO 31000 ו-ISO 45001

- **ל-31000:** 31000 מגדיר את תהליך ניהול-הסיכונים; 31010 הוא ארגז-הכלים של שלב
  ההערכה בתוכו (זיהוי → ניתוח → הערכה). ראה [`ISO-31000-DRAFT.md`](ISO-31000-DRAFT.md).
- **ל-45001:** מערכת ניהול-הבטיחות לפי 45001 דורשת זיהוי-מפגעים והערכת-סיכונים שיטתיים
  (סעיף 6.1). השיטות שכאן הן ה**כיצד** המעשי שממלא דרישה זו, ומזינות את **מדרג-הבקרות**
  (scope 5.6) ואת **התכנית-לניהול-הבטיחות** (scope 1.2 — תקנות תכנית לניהול הבטיחות,
  תשע"ג-2013).

**מקרא:** ✅ נלמד ומכוסה בקורס · 🟠 מכוסה חלקית · 🔴 טיוטה חדשה (scope 5.3, ממתין
להכרעת-מוטי על שילוב ומיקום סופי).
