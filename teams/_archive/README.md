> כאן נשמרים סוכנים שהוסרו מהצוות-הפעיל. **לא מוחקים זיכרון לתמיד — מארכבים.**
> זהו הזיכרון-הארגוני של מה-שלא-עבד. ראה פרוטוקול מחזור-החיים ב-[`../ORG.md`](../ORG.md).

---

## מתי סוכן מגיע לכאן

המתווך (אמיר) הכריע **"החלפה"** (ולא "שיקום") בעקבות בקרת-סחף — למשל סוכן
שחרג שוב-ושוב מ-`identity.md`, הפר red-line, או נכשל ב-loop-breaker. ראה
§בקרת-סחף ו§מחזור-חיים ב-[`../ORG.md`](../ORG.md).

## מבנה תיקיית-ארכוב

```
_archive/
  <slug>-<YYYY-MM-DD>/
    identity.md         ← הזהות שהיתה (כפי-שהיא, ללא עריכה)
    memory.md           ← הזיכרון הסורר (לקריאה/לקח בלבד — לא יורש לסוכן-חדש)
    activity-log.md     ← יומן-הפעילות עד-ההסרה
    REASON.md           ← למה הוסר: סוג-הסטייה · מי הכריע · תאריך · ראיה
```

> **חשוב:** סוכן-חדש שמחליף את הסורר מתחיל עם `memory.md` **חדש ונקי** —
> הוא **אינו** יורש את הזיכרון מהארכוב (כדי לא לרשת את דפוס-הסחף).

## תבנית `REASON.md`

```md
# הסרת `<slug>` — <YYYY-MM-DD>

- **סוג-הסטייה:** <חריגת-Scope / הפרת-red-line / loop / תוכן-מומצא / אחר>
- **מי הכריע:** mediator (אמיר) · אושר ע"י: <מועצה/מוטי אם נדרש>
- **ראיה:** <קישור ל-activity-log.md + שדה ה-Self-check / control-report.md>
- **החלטה:** ארכוב + <החלפה בסוכן-חדש / סגירת-התפקיד>
- **לקח-ארגוני:** <מה לומדים כדי שלא יחזור>
```
