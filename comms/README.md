# comms/ — פרוטוקול תקשורת בין-סוכנית

תקשורת בין סוכנים מבוססת-**קבצים**, לא ישירה בזיכרון. כך נמנעים race
conditions ו-deadlocks, ומתאפשרת **התאוששות-מקריסה** (כל שלב נשמר על הדיסק).

## איך זה עובד
1. סוכן ששולח מסר כותב שורת **JSONL append-only** לקובץ ב-`comms/`.
2. כל שורה כוללת תגית-ניתוב: `<<SEND_MESSAGE>>` (לסוכן יחיד) או
   `<<BROADCAST>>` (לכל הצוות).
3. האורקסטרטור/`gesher` מאזין א-סינכרונית, מפרסר, וטוען את המסר להקשר-היעד.

## פורמט שורה
```json
{"ts": "<ISO-8601>", "from": "<slug>", "to": "<slug|*>",
 "tag": "SEND_MESSAGE", "body": "<תוכן>"}
```

## כללים
- **append-only** — לעולם לא דורסים שורה קיימת (מונע התנגשות).
- כתיבה מסודרת בתור — שני סוכנים לא דורסים זה את זה.
- **Resume:** לאחר קריסה, המערכת קוראת את ה-JSONL וממשיכה מהנקודה האחרונה.
- אין תקשורת ישירה peer-to-peer בזיכרון.
